# Идеи для улучшений

## 1) Автоматическая валидация структуры входных данных MOEX
- **Задача:** добавить проверку, что ключевые колонки (`SECID`, `ISIN`, `EMITENT_ID`, `STATUS`) действительно пришли из API, и понятное сообщение об ошибке, если структура изменилась.
- **Зачем это нужно:** MOEX может менять формат ответа, и это лучше ловить сразу.
- **Профит простыми словами:** меньше «тихих» ошибок и неожиданных пустых отчетов.
- **Степень полезности:** 5/5.

## 2) История изменений по эмитентам
- **Задача:** хранить отдельный журнал, когда впервые обнаружен новый эмитент (дата + EMITENT_ID + название).
- **Зачем это нужно:** удобнее отслеживать динамику появления новых компаний в выборке.
- **Профит простыми словами:** можно быстро понять, какие новые эмитенты добавились за период.
- **Степень полезности:** 4/5.

## 3) Контроль размера кэша
- **Задача:** добавить мягкий лимит на папку `cache/moex` и удаление самых старых записей при превышении порога.
- **Зачем это нужно:** длительная эксплуатация может раздувать диск.
- **Профит простыми словами:** проект дольше остается «легким» без ручной чистки.
- **Степень полезности:** 4/5.
