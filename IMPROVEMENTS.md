# IMPROVEMENTS

## Идея 1
- **Задача:** Оценить переход на SQLite для хранения кэша загрузки, исключений и результатов скоринга.
- **Зачем:** JSON-чекпоинты простые, но SQLite даст транзакции, индексы и устойчивость при больших объемах данных.
- **Профит простыми словами:** Больше надежности при сбоях и быстрее поиск/обновление по конкретной бумаге.
- **Полезность 1..5:** 5

## Идея 2
- **Задача:** Добавить режим `storage_backend: json|sqlite` в `config.yml`.
- **Зачем:** Чтобы сохранить простоту старта (JSON по умолчанию), но дать путь к масштабированию без переписывания всей логики.
- **Профит простыми словами:** Можно включить SQLite одной настройкой, когда проект вырастет.
- **Полезность 1..5:** 5

## Идея 3
- **Задача:** Добавить таблицу `runs` (SQLite) с историей запусков и метриками ошибок/времени.
- **Зачем:** Нужна диагностика деградации качества сети и производительности не только по `latest.log`.
- **Профит простыми словами:** Пользователь видит, когда и почему запуск стал идти дольше.
- **Полезность 1..5:** 5
