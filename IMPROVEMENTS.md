# IMPROVEMENTS

## Идея 11
- **Задача:** Добавить автоматическую подсказку по lock-файлам (`*.lock`) в конце запуска (`что найдено` / `как снять`).
- **Зачем:** Пользователь может забыть удалить lock-файл и не понять, почему изменился режим.
- **Профит простыми словами:** Меньше «магии», проще поддержка без погружения в код.
- **Полезность 1..5:** 4

## Идея 12
- **Задача:** Добавить Excel-лист `EXCLUSIONS_HISTORY` с историей исключений из `state/exclusions_history.json`.
- **Зачем:** Сейчас история есть в JSON, но пользователю удобнее смотреть её в привычном Excel.
- **Профит простыми словами:** История причин исключений становится доступной в 2 клика.
- **Полезность 1..5:** 5

## Идея 13
- **Задача:** Добавить режим `run.py` «только отчёт» без сетевых запросов (на основе сохранённых state + runs).
- **Зачем:** Для быстрой проверки результатов не всегда нужен полный повторный парсинг MOEX.
- **Профит простыми словами:** Можно мгновенно получить сводку и файлы без ожидания сети.
- **Полезность 1..5:** 4

## Идея 14
- **Задача:** Добавить проверку качества `emitents.xlsx`: доля строк с `warning` и аварийный WARN в логе при превышении порога.
- **Зачем:** Пустые ИНН/наименования лучше отслеживать сразу после запуска, а не в ручной проверке.
- **Профит простыми словами:** Быстро видно, когда источник данных «просел» по качеству.
- **Полезность 1..5:** 5

## Идея 15
- **Задача:** Добавить мини-утилиту `scripts/show_emitents_gaps.py` (топ проблемных эмитентов: пустой ИНН, пустое имя, нет акций/облигаций).
- **Зачем:** Сейчас такие проблемы видны только вручную фильтрами в Excel.
- **Профит простыми словами:** Экономия времени на контроль качества данных.
- **Полезность 1..5:** 4

## Идея 16
- **Задача:** Добавить fallback-парсер `description` MOEX для разных регистров колонок (`name/value`, `NAME/VALUE`) и логировать формат ответа.
- **Зачем:** Формат блока `description` может меняться по регистру, из-за чего пропадают `full_name`/`ИНН` без явной ошибки.
- **Профит простыми словами:** Справочник эмитентов не «пустеет» из-за мелких изменений API.
- **Полезность 1..5:** 5

## Идея 17
- **Задача:** Добавить метрику в SUMMARY: сколько эмитентов взято из кэша и сколько дозагружено через `emitters/{EMITTER_ID}`.
- **Зачем:** Сейчас видно только общее время, но не видно реального эффекта кэша по эмитентам.
- **Профит простыми словами:** Пользователь сразу понимает, почему повторный запуск быстрый или медленный.
- **Полезность 1..5:** 5

## Идея 18
- **Задача:** Добавить в этап эмитентов heartbeat-лог каждые N секунд (например, каждые 5с) с текущим `processed/total`.
- **Зачем:** При медленной сети пользователь иногда воспринимает длительный этап как зависание.
- **Профит простыми словами:** Видно, что процесс живой и двигается даже при редких обновлениях прогресса.
- **Полезность 1..5:** 5


## Идея 19
- **Задача:** Добавить отдельный lightweight-кэш по тикерам акций (`emitter_id -> [SECID]`) без хранения полного JSON рынка shares.
- **Зачем:** Даже с кэшем полная структура ответа рынка занимает лишнее место и медленнее читается из SQLite/JSON.
- **Профит простыми словами:** Быстрее старт и меньше размер состояния.
- **Полезность 1..5:** 5

## Идея 20

- **Зачем:** Сейчас ускорения оцениваются по ощущениям, без стабильной метрики в репозитории.

- **Полезность 1..5:** 4

## Идея 21
- **Задача:** Добавить отдельный кэш `market_secid_description` (TTL 7 дней), чтобы не дергать `description` повторно для уже известных `SECID` из `shares/bonds`.
- **Зачем:** При полном обходе MOEX список `SECID` большой, и даже с текущим кэшем полезно разделить «быстрый» market-кэш и «долгий» кэш сопоставлений.
- **Профит простыми словами:** Повторные запуски будут еще быстрее и стабильнее по времени.
- **Полезность 1..5:** 5

## Идея 24
- **Задача:** Добавить heartbeat в фазу `market_descriptions`: печатать промежуточный статус каждые 5 секунд даже при отсутствии завершённых запросов.
- **Зачем:** При медленном API первый завершившийся запрос может появиться не сразу, и пользователю кажется, что скрипт завис.
- **Профит простыми словами:** Видно, что процесс идёт, даже если сеть отвечает медленно.
- **Полезность 1..5:** 5
