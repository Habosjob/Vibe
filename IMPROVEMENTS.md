# Идеи для улучшений

## 1) Режим «только изменения с прошлой выгрузки»
- **Задача:** добавить сравнение текущего набора облигаций с прошлым запуском и догружать только новые/изменившиеся бумаги.
- **Зачем это нужно:** чтобы не тратить время на повторную обработку полностью одинаковых данных.
- **Профит простыми словами:** выгрузка станет заметно быстрее в обычные дни.
- **Степень полезности:** 5/5.

## 2) Умный повтор только проваленных SECID
- **Задача:** вести отдельный список бумаг, по которым была ошибка сети, и в конце делать дополнительный «добор» только по ним.
- **Зачем это нужно:** чтобы единичные сетевые ошибки не портили итог.
- **Профит простыми словами:** меньше ручных перезапусков и более полный результат.
- **Степень полезности:** 5/5.

## 3) Сжатие сырых JSON в архив после успешного запуска
- **Задача:** после успешной выгрузки архивировать `raw/*.json` в zip и хранить ограниченное число архивов.
- **Зачем это нужно:** сократить место на диске при сохранении отладочной истории.
- **Профит простыми словами:** папка проекта не разрастается, но при необходимости есть что посмотреть.
- **Степень полезности:** 4/5.

## 4) Отдельный лист «Качество данных»
- **Задача:** добавить сводку с количеством пустых полей по ключевым колонкам (`SECID`, `ISIN`, `EMITENT_ID` и т.д.).
- **Зачем это нужно:** быстро видеть, где у источника были неполные данные.
- **Профит простыми словами:** проще контролировать качество перед использованием отчёта.
- **Степень полезности:** 4/5.
