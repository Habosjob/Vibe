# Improvement backlog

## 1) История изменений по выпускам облигаций (отдельный лист HISTORY)
- **Задача:** Накапливать изменения ключевых полей (купон, цена, статус торгов) и выводить их в отдельный лист `HISTORY`.
- **Зачем это нужно:** Позволит видеть динамику, а не только «срез на сейчас».
- **Польза простым языком:** Можно понять, что изменилось со вчерашнего дня, не сравнивая файлы вручную.
- **Степень полезности:** **5/5**

## 2) Автоматическая проверка качества данных перед выгрузкой
- **Задача:** Добавить этап валидации (пустые ISIN, дубликаты, странные значения купона) и краткий отчёт в лог.
- **Зачем это нужно:** Снизит риск передачи «грязных» данных в отчёт.
- **Польза простым языком:** Меньше шансов получить ошибку в аналитике из-за случайно плохих данных.
- **Степень полезности:** **5/5**

## 3) Сравнение текущей выгрузки с прошлой и подсветка отличий в Excel
- **Задача:** При сохранении Excel визуально отмечать новые/изменённые строки цветом.
- **Зачем это нужно:** Быстрое визуальное понимание изменений без ручного сравнения.
- **Польза простым языком:** Открыл файл — сразу видно, что новое и что поменялось.
- **Степень полезности:** **4/5**

## 4) Чекпоинты по полному циклу выгрузки (не только эмитенты)
- **Задача:** Добавить пошаговые чекпоинты для всех этапов (получение списка бумаг, обогащение, запись Excel), чтобы можно было продолжить не с начала.
- **Зачем это нужно:** Сейчас чекпоинты есть только для обогащения эмитентов; полный контур даст ещё больше устойчивости.
- **Польза простым языком:** Если интернет пропал в середине, скрипт продолжит с ближайшего шага, а не начнёт всё заново.
- **Степень полезности:** **5/5**
