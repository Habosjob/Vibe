# Сбор облигаций MOEX в Excel

## Что делает проект
Скрипт получает данные по облигациям с MOEX, фильтрует выпуски по сроку до погашения, обогащает данные информацией об эмитенте и формирует Excel-отчёт.

Сценарий запускается напрямую из VS Code: **без `argparse` и без параметров командной строки**.

## Основной файл
- `moex_bonds_to_excel.py` — главный скрипт выгрузки.

## Что происходит по шагам
1. Создаются папки `logs`, `raw`, `cache`, `output`.
2. Папка `raw` очищается перед запуском.
3. Лог `logs/moex_bonds.log` перезаписывается (всегда только последний запуск).
4. Читается основной кэш выгрузки.
5. Запрашиваются свежие бумаги с MOEX.
6. Дополнительно парсятся рыночные данные: `VOLTODAY`, `VALTODAY`, `NUMTRADES`, `YIELD`.
7. Исключаются бумаги, у которых до погашения меньше 1 года.
8. Загружаются суточные метрики и отмечаются бумаги, где амортизация уже началась или начнётся менее чем через 1 год.
9. Выполняется первичная фильтрация по данным MOEX: по датам погашения, амортизации и оферт.
10. Запускается обогащение из внешних источников (DOHOD/Corpbonds) и данных эмитента.
11. Проверка оферт и формул купона идёт параллельно, с лимитом потоков.
12. Для выпусков с `COUPONVALUE=0` рассчитывается `COUPONPERCENT` по формуле (с fallback на Corpbonds).
13. Для расчётного купона в поле `COUPON_FORMULA_SOURCE` указывается источник: `DOHOD` или `CORPBONDS`.
14. Для оферт используется отдельный 7-дневный кэш `cache/offer_verification_cache.json`; при изменении версии формата старый кэш автоматически удаляется и собирается заново.
15. Для расчёта купона используется отдельный 7-дневный кэш `cache/coupon_formula_cache.json`.
16. Добавляются данные эмитента (`ISSUER_NAME`, `ISSUER_INN`, `ISSUER_BOND_CLASS`), признак `QUALIFIED_INVESTOR` и `BOND_TYPE`.
17. Структурные облигации исключаются из итогового отчёта.
18. Если `COUPONPERIOD = 0`, период пытается восстановиться из `COUPONFREQUENCY`.
19. Если после восстановления `COUPONPERIOD` всё равно `0`, бумага исключается.
20. После обогащения снова применяется фильтрация по офертам и амортизации.
21. Выполняется проверка качества данных.
22. В проверке качества отдельно выводится контроль оферт: сколько бумаг с офертами, сколько из них без даты оферты и сколько с совпадением даты оферты и даты погашения.
23. Сохраняются `raw/moex_bonds_raw.json` и `output/moex_bonds.xlsx`.
24. В лог записывается время выполнения.

## Что в Excel
### Лист `MOEX_BONDS`
- Колонки сгруппированы цветными секциями:
  - `Эмитент`: `ISSUER_NAME`, `ISSUER_INN`, `ISSUER_BOND_CLASS`, `SHORTNAME`.
  - `Квалификация и тип`: `QUALIFIED_INVESTOR`, `BOND_TYPE`.
  - `Оферты`: `HAS_PUT_CALL_OFFER`, `PUT_CALL_OFFER_DATE`.
  - `Торги, погашение и амортизация`: `HAS_AMORTIZATION`, `AMORTIZATION_START_DATE`, `MATDATE`.
  - `Купоны`: `COUPONVALUE`, `ACCRUEDINT`, `COUPONPERIOD`, `COUPONPERCENT`, `COUPON_FORMULA_SOURCE`.
  - `Рынок`: `FACEVALUE`, `FACEUNIT`, `PRIMARYBOARDID`, `PREVLEGALCLOSEPRICE`, `PREVPRICE`, `VOLTODAY`, `VALTODAY`, `NUMTRADES`, `YIELD`.
- `COUPON_FORMULA_SOURCE` — источник формулы для расчётного `COUPONPERCENT` (`DOHOD` или `CORPBONDS`).
- Расчётные значения `COUPONPERCENT` подсвечиваются жёлтым.
- `SECID` скрыт (для технических связок).

### Лист `INFO`
Пояснение полей простым русским языком.

## Структура папок
- `logs/` — лог последнего запуска.
- `raw/` — сырые данные для отладки (очищается автоматически).
- `cache/` — кэши (основной, пожизненный по эмитентам/`COUPONPERIOD`, суточный по метрикам, кэш оферт, кэш формул купона).
- `output/` — итоговый Excel.

## Как запустить
```bash
python moex_bonds_to_excel.py
```

После запуска проверьте:
- `output/moex_bonds.xlsx`
- `logs/moex_bonds.log`

## Служебные файлы
- `IMPROVEMENTS.md` — идеи для развития.
- `CHANGELOG.md` — история реализованных изменений.
