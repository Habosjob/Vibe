# Сбор облигаций MOEX в Excel

## Что делает проект
Этот проект получает список облигаций с Московской биржи (MOEX) и формирует удобный Excel-файл для ежедневной работы.

Скрипт рассчитан на простой запуск в VS Code: **без аргументов командной строки**.

---

## Основной файл
- `moex_bonds_to_excel.py` — главный скрипт выгрузки и подготовки отчёта.

### Что делает скрипт по шагам
1. Создаёт служебные папки (`logs`, `raw`, `cache`, `output`).
2. Полностью очищает папку `raw` перед запуском.
3. Перезаписывает лог `logs/moex_bonds.log` (лог всегда только за последний запуск).
4. Пытается прочитать кэш.
5. Запрашивает свежие данные с MOEX.
6. Выполняет инкрементальное обновление (добавляет новые записи и обновляет изменившиеся).
7. Сохраняет сырые данные в `raw/moex_bonds_raw.json`.
8. Запускает автоматическую проверку качества данных (пустые ISIN, дубликаты, аномальные купоны) и пишет результат в лог.
9. Обогащает данные: добавляет наименование эмитента (`ISSUER_NAME`) и ИНН эмитента (`ISSUER_INN`).
10. Для эмитентов использует пожизненный справочник и пакетный режим, а также пишет чекпоинты на случай обрывов.
11. Формирует `output/moex_bonds.xlsx` с улучшенным форматированием и листом `INFO`.
12. В конце пишет общее время выполнения.

## Что в итоговом Excel
### Лист `MOEX_BONDS`
- Основные поля по облигациям.
- Колонка `ISIN` автоматически переносится на первое место.
- Колонка `ISSUER_NAME` автоматически ставится сразу после `ISIN`.
- Добавлена колонка `ISSUER_INN` с ИНН эмитента.
- Включены фильтры по заголовкам.
- Шапка заморожена.
- Ширина колонок подогнана автоматически, для `ISSUER_NAME` задана расширенная ширина для длинных названий.
- Улучшен внешний вид: отключены линии сетки, добавлена «зебра» строк и формат чисел для финансовых полей.
- Колонка `SECID` **скрыта** (она есть в файле, но не мешает при открытии).
- Колонки `SECNAME`, `LISTLEVEL`, `STATUS` удалены по требованиям.

### Лист `INFO`
- Пояснение всех ключевых полей простым языком.

## Структура папок
- `logs/` — лог последнего запуска.
- `raw/` — сырые данные для отладки (очищается автоматически перед каждым запуском).
- `cache/` — кэш API-ответов, пожизненный справочник эмитентов и checkpoint-файл восстановления.
- `output/` — итоговый Excel-файл.

---

## Как запустить
Откройте проект в VS Code и выполните:

```bash
python moex_bonds_to_excel.py
```

После завершения проверьте:
- `output/moex_bonds.xlsx`
- `logs/moex_bonds.log`
