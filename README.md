# Сбор облигаций MOEX в Excel

## Что делает проект
Скрипт получает данные по облигациям с MOEX, фильтрует выпуски по сроку до погашения, обогащает данные информацией об эмитенте и формирует Excel-отчёт.

Сценарий запускается напрямую из VS Code: **без `argparse` и без параметров командной строки**.

---

## Основной файл
- `moex_bonds_to_excel.py` — главный скрипт выгрузки.

## Что происходит по шагам
1. Создаются папки `logs`, `raw`, `cache`, `output`.
2. Папка `raw` очищается перед запуском.
3. Лог `logs/moex_bonds.log` перезаписывается (всегда только последний запуск).
4. Читается основной кэш выгрузки.
5. Запрашиваются свежие бумаги с MOEX.
6. Дополнительно парсится блок рыночных данных: объём торгов (`VOLTODAY`), оборот (`VALTODAY`) и число сделок (`NUMTRADES`).
7. Исключаются бумаги, у которых до погашения меньше 1 года.
8. Загружаются суточные метрики и отмечаются бумаги, где амортизация уже началась или начнётся менее чем через 1 год.
9. Дополнительно исключаются бумаги, у которых дата оферты наступит раньше, чем через 1 год от текущей даты.
10. Такие бумаги исключаются из итогового отчёта и запоминаются в суточном кэше (чтобы не опрашивать их повторно каждый запуск).
11. Выполняется проверка качества данных.
12. Добавляются данные эмитента (`ISSUER_NAME`, `ISSUER_INN`, `ISSUER_SECTOR`, `ISSUER_BOND_CLASS`), признак `QUALIFIED_INVESTOR` и тип облигации `BOND_TYPE`.
13. Структурные облигации полностью исключаются из итогового отчёта и помечаются в пожизненном кэше, чтобы не определять этот тип заново на каждом запуске.
14. Если `COUPONPERIOD = 0`, период пытается восстановиться из `COUPONFREQUENCY` и запоминается в пожизненном кэше.
15. Если после восстановления `COUPONPERIOD` всё равно `0`, такая бумага исключается из итогового отчёта.
16. Парсятся оферты из MOEX (`offers`): `HAS_PUT_CALL_OFFER` и `PUT_CALL_OFFER_DATE`.
17. В отчёте учитываются только **актуальные** оферты (с датой сегодня или в будущем). Прошедшие оферты не считаются активными.
18. Если у бумаги несколько оферт, берётся ближайшая актуальная дата.
19. Для данных эмитента используется **пожизненный кэш**.
20. Даты в итоге приводятся к формату `ДД.ММ.ГГГГ`.
21. Сохраняются `raw/moex_bonds_raw.json` и `output/moex_bonds.xlsx`.
22. В лог записывается время выполнения.

## Что в итоговом Excel
### Лист `MOEX_BONDS`
- `ISIN` — в первой колонке.
- Добавлены столбцы-разделители между блоками данных. Название каждой группы теперь пишется вертикально прямо в разделителе (объединение ячеек 1-2 строки), чтобы при сворачивании макет оставался читаемым.
- Теперь каждый блок колонок идёт подряд и имеет свою кнопку сворачивания/разворачивания. Разделители сделаны шире и окрашиваются разными оттенками, чтобы соседние блоки визуально не сливались:
  - `Эмитент`: `ISSUER_NAME`, `ISSUER_INN`, `ISSUER_SECTOR`, `ISSUER_BOND_CLASS`, `SHORTNAME`.
  - `Квалификация и тип`: `QUALIFIED_INVESTOR`, `BOND_TYPE`.
  - `Оферты`: `HAS_PUT_CALL_OFFER`, `PUT_CALL_OFFER_DATE`.
  - `Погашение и амортизация`: `HAS_AMORTIZATION`, `AMORTIZATION_START_DATE`, `MATDATE`.
  - `Купоны`: `COUPONVALUE`, `ACCRUEDINT`, `COUPONPERIOD`, `COUPONPERCENT`.
  - `Рынок`: `FACEVALUE`, `FACEUNIT`, `PRIMARYBOARDID`, `PREVLEGALCLOSEPRICE`, `PREVPRICE`, `VOLTODAY`, `VALTODAY`, `NUMTRADES`.
- `QUALIFIED_INVESTOR` — ✔/✖.
- `ISSUER_SECTOR` — отраслевой сектор эмитента (например: банки и финансовые институты, машиностроение, нефтегаз и т.д.) по названию компании.
- `ISSUER_BOND_CLASS` — тип бумаги на рынке облигаций (государственный, корпоративный, муниципальный/субфедеральный, иностранный).
- `BOND_TYPE` — тип облигации (например, фикс/флоатер). Структурные облигации в итоговый лист не попадают.
- `HAS_PUT_CALL_OFFER` — ✔/✖ (есть ли Put/Call оферта).
- `PUT_CALL_OFFER_DATE` — ближайшая дата Put/Call оферты (если есть).
- `HAS_AMORTIZATION` — ✔/✖ (финальное погашение не считается амортизацией).
- `AMORTIZATION_START_DATE` — дата старта амортизации (если есть).
- `ACCRUEDINT` — НКД.
- `MATDATE` — дата погашения (формат `ДД.ММ.ГГГГ`).
- `VOLTODAY` — объём торгов за день (количество бумаг).
- `VALTODAY` — оборот торгов за день (в деньгах).
- `NUMTRADES` — количество сделок за день.
- `SECID` скрыт (для технических связок).
- Колонки `SECNAME`, `LISTLEVEL`, `STATUS` удалены.

### Лист `INFO`
Пояснение полей на простом русском языке.

## Структура папок
- `logs/` — лог последнего запуска.
- `raw/` — сырые данные для отладки (очищается автоматически).
- `cache/` — кэши (основной, пожизненный по эмитентам и `COUPONPERIOD`, суточный по метрикам с версией схемы, амортизацией и офертами).
- `output/` — итоговый Excel.

---

## Как запустить
```bash
python moex_bonds_to_excel.py
```

После запуска проверьте:
- `output/moex_bonds.xlsx`
- `logs/moex_bonds.log`
