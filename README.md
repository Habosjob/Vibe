# Скрипт выгрузки облигаций MOEX + CorpBonds

Скрипт загружает все торгуемые облигации MOEX, считает динамику цены и YTM, отдельно парсит карточки CorpBonds по `SECID` и сохраняет результаты в Excel.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Что делает скрипт

1. Загружает список облигаций с MOEX (`is_traded=1`).
2. Подтягивает историю объёма за 20 дней.
3. Загружает описание бумаги, эмитента и амортизации из MOEX.
4. Для всех `SECID` парсит страницу `https://corpbonds.ru/bond/{SECID}`.
5. Из CorpBonds сохраняет: цена, рейтинг, тип купона, формула купона, дата ближайшей оферты, купон лесенкой.
6. Рассчитывает YTM:
   - цена для расчёта берётся с приоритетом MOEX, если у MOEX пусто — из CorpBonds;
   - НКД всегда учитывается в грязной цене;
   - для флоатеров и ОФЗ 50* применяется прогноз индексов из `config.py` + формула купона (если парсится);
   - в приоритете используется `YIELD` от MOEX (если значение валидное), а при его отсутствии — резервная оценка на основании купона, цены и срока до оферты/погашения.
7. Формирует объединенный файл `MergeBonds.xlsx` по `Secid`:
   - `Цена` из CorpBonds сохраняется как `Цена CorpBonds`.
   - `Рейтинг`, `Тип купона`, `Формула купона`, `Купон лесенкой` переносятся как есть.
   - `Дата ближайшей оферты` из CorpBonds объединяется с `Дата оферты` из MOEX:
     - если у MOEX дата есть — не меняем;
     - если у MOEX дата пустая — подставляем дату из CorpBonds.
8. Сохраняет данные в SQLite (инкрементально + кэш по TTL).
9. Формирует Excel-файлы:
   - `output/MoexBonds.xlsx`
   - `output/CorpBonds.xlsx`
   - `output/MergeBonds.xlsx`

## Где менять настройки

Все параметры находятся в `config.py`:

- пути проекта;
- таймауты/ретраи/параллельность;
- прогнозы индексов для YTM (`YTM_KEY_RATE_FORECAST`, `YTM_INFLATION_FORECAST`);
- скорость и кэш CorpBonds (`CORPBONDS_MAX_CONCURRENT_TASKS`, `CORPBONDS_CACHE_TTL_SEC`);
- TTL кэша;
- флаги выгрузки:
  - `EXPORT_MOEX_TO_EXCEL`
  - `EXPORT_CORPBONDS_TO_EXCEL`
- имена Excel файлов и листов.

## Выходные папки

- `output/` — Excel-файлы выгрузки;
- `logs/` — логи с ротацией;
- `db/` — база SQLite (снимок цен + кэши MOEX/CorpBonds);
- `cache/` — состояние выполнения `state.json`;
- `raw/` — резерв под сырые данные.

## Частые проблемы

1. `ModuleNotFoundError` — не установлены зависимости.
2. Пустые поля CorpBonds — для части бумаг сайт может не отдавать нужные строки или временно недоступен.
3. Долгая работа — уменьшите `MAX_CONCURRENT_TASKS`.
   Для ускорения этапа CorpBonds меняйте `CORPBONDS_MAX_CONCURRENT_TASKS`.
4. Нет динамики цены — первый запуск, в БД ещё нет предыдущей цены.
5. При сетевых ошибках смотрите `logs/app.log`.
