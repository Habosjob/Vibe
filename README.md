# Vibe — утилиты для работы с данными

## Что есть в проекте

### 1) `moex_bonds_export.py`
Скрипт для выгрузки **торгуемых облигаций MOEX** в Excel, вместе с:
- основными параметрами выпусков,
- расширенными данными по каждой бумаге (описание, режимы торгов, купоны, амортизации, оферты),
- данными по эмитентам (ID, название, ИНН, ОКПО, количество выпусков и список кодов бумаг).

#### Куда сохраняется результат
- Excel: `output/moex_bonds_full_export.xlsx`
- Лог запуска: `logs/moex_bonds_export.log`
- Сырые данные для отладки: папка `raw/`

#### Как запускать (через VS Code)
1. Откройте проект в VS Code.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Запустите файл `moex_bonds_export.py` кнопкой **Run Python File**.

#### Важные особенности
- Лог-файл перезаписывается при каждом запуске (не разрастается).
- Папка `raw/` очищается перед новым запуском и заполняется заново (служебный `.gitkeep` сохраняется).
- Используется кэш в `cache/moex` для ускорения повторных запусков:
  - кэш карточек отдельных облигаций,
  - кэш справочных строк по эмитентам в `cache/moex/reference_rows/`.
- Расширенные данные по бумагам и справочные данные по эмитентам собираются параллельно в несколько потоков.
- В консоли выводятся этапы выполнения и прогресс, чтобы не было ощущения «зависания».
- Убрана проблема «бесконечного цикла» по рынку облигаций: вместо длинной прокрутки всего `/securities` скрипт делает точечные запросы только по реально найденным облигациям.

#### Почему запуск может быть не мгновенным
Скрипт обращается к MOEX API по каждой найденной облигации, поэтому первый запуск может занять время. Повторные запуски быстрее за счёт кэша.

---

### 2) `generate_raw_links.py`
Служебный скрипт, который формирует списки RAW-ссылок для файлов репозитория GitHub.

## Структура рабочих папок
- `logs/` — логи последнего запуска.
- `raw/` — сырые JSON-данные для отладки.
- `cache/` — кэш ответов MOEX API.
- `output/` — итоговые Excel-файлы.

## Для не‑технического пользователя
Если вам нужен готовый файл с облигациями и эмитентами, запускайте только `moex_bonds_export.py`. Остальное в проекте — вспомогательные материалы.
