# Vibe — утилиты для работы с данными

## Что есть в проекте

### 1) `moex_bonds_export.py`
Главный скрипт для выгрузки **торгуемых облигаций MOEX**.

Теперь выгрузка устроена так, чтобы работать стабильнее и быстрее при повторных запусках:
- загрузка расширенных данных делится на **10 частей (чанков)**;
- после каждого чанка сохраняется **checkpoint** (можно продолжить после сбоя сети);
- срок жизни кэша вынесен в отдельный файл `moex_bonds_config.py`;
- итог разбит на несколько Excel-файлов, чтобы основной файл не был слишком тяжёлым;
- лист `bond_descriptions` переведен в широкий формат: **1 строка = 1 облигация**.

### 2) `moex_bonds_config.py`
Файл настроек с комментариями на русском языке.

Здесь можно без программиста поменять:
- количество потоков;
- таймауты и повторы запросов;
- срок хранения разных типов кэша;
- количество чанков;
- лимит облигаций для тестового запуска.

### 3) `generate_raw_links.py`
Служебный скрипт для формирования RAW-ссылок файлов репозитория GitHub.

---

## Какие файлы формируются

### Основной Excel
- `output/moex_bonds_full_export.xlsx`
  - `bonds_traded`
  - `bond_descriptions` (читабельный широкий формат)

### Отдельные справочники
- `output/moex_emitents.xlsx`
- `output/moex_bond_coupons.xlsx`
- `output/moex_bond_amortizations.xlsx`
- `output/moex_bond_offers.xlsx`

### Технические данные
- Лог запуска: `logs/moex_bonds_export.log` (каждый запуск перезаписывает файл)
- Сырые данные для отладки: `raw/` (очищается перед запуском)
- Кэш и checkpoint: `cache/moex/`

## Как запускать (через VS Code)
1. Откройте проект в VS Code.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. При необходимости измените настройки в `moex_bonds_config.py`.
4. Запустите `moex_bonds_export.py` кнопкой **Run Python File**.

## Для пользователя, далёкого от IT
Если нужен только итоговый отчёт — запускайте `moex_bonds_export.py`.
Скрипт сам очистит старые временные данные, обновит информацию и соберёт Excel-файлы в папке `output/`.
