# MOEX Bonds Loader (Этап 2)

Скрипт загружает актуальные облигации MOEX через ISS API, считает ликвидность за 10 торговых дней, сохраняет витрину в SQLite и (опционально) в Excel.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Где менять настройки

Все параметры находятся в `config.py`:

- сетевые таймауты, retry/backoff;
- ограничение параллельности;
- пути к папкам/БД;
- включение/выключение Excel;
- параметры расчёта ликвидности;
- политика исключения погашенных бумаг.

## Что делает скрипт

1. Получает список торгуемых облигаций (MOEX stock/bonds).
2. Для каждой бумаги догружает:
   - карточку бумаги и `EMITTER_ID`;
   - данные эмитента из CCI (название, ИНН);
   - рейтинги эмитента из CCI;
   - marketdata (цена LAST/CLOSEPRICE);
   - history (ликвидность за последние 10 торговых дней из окна 20 календарных);
   - графики coupons/amortizations (bondization).
3. Сохраняет данные в SQLite с upsert (инкрементально).
4. Обновляет checkpoint: `cache/state.json`.
5. Печатает итоговый Summary по запуску.

## Папки и файлы

- `db/bonds.sqlite3` — основная база данных.
- `cache/state.json` — чекпоинт выполнения.
- `logs/app.log` — лог с ротацией.
- `output/MoexBonds.xlsx` — итоговая выгрузка (если `EXPORT_EXCEL=True`).
- `output/MoexBonds.locked.xlsx` — резервный файл, если основной Excel открыт и заблокирован.

## Что безопасно удалять

- `cache/`, `raw/`, `logs/` — безопасно.
- `db/` — удаляйте только если хотите полностью сбросить историю.

Для полной очистки используйте:

```bash
python erase_data.py
```

## Частые проблемы

1. `MoexBonds.xlsx` не перезаписывается — файл открыт в Excel. Закройте файл и перезапустите.
2. Ошибки сети/таймаута MOEX — скрипт делает retry; детали в `logs/app.log`.
3. Пустые поля по части бумаг — ISS может не отдавать некоторые атрибуты; это логируется как `missing_count`.
4. Медленная обработка — уменьшите/увеличьте `MAX_CONCURRENCY` в `config.py`.
5. Ошибка конфигурации — проверьте значения в `config.py` (диапазоны и типы).
