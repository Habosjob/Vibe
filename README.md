# Vibe — утилиты для работы с данными облигаций MOEX

## Что делает проект

### 1) `moex_bonds_export.py`
Главный скрипт, который собирает данные по облигациям MOEX и формирует готовые файлы.

Сейчас логика такая:
- берутся только торгуемые облигации;
- листы `bonds_traded` и `bond_descriptions` объединяются в один итоговый лист `bonds_traded`;
- в итоговый лист `bonds_traded` дополнительно подставляется `Дата амортизации` из справочника амортизаций (берётся ближайшая будущая амортизация, а не дата финального погашения), если биржа передала это значение;
- дубли полей при объединении автоматически удаляются (включая дубли с разным регистром и лишними пробелами);
- лишние столбцы из бизнес-списка автоматически исключаются из итогового отчета;
- заголовки торговых колонок переименовываются на понятные русские названия;
- справочники сохраняются в формате **Parquet** (быстрее и удобнее для машинной обработки, чем Excel);
- справочник `moex_emitents` обновляется только новыми эмитентами (старые строки не перезаписываются);
- расширенные данные грузятся параллельно, чанками, с checkpoint и кэшем;
- в листе `bonds_traded` колонки сгруппированы по жизненному циклу облигации (идентификация → эмитент → параметры выпуска → купоны → погашение/оферты → рыночные данные), добавлены цветные столбцы-разделители с понятными названиями групп, а группы можно сворачивать и разворачивать в Excel.
- добавлен этап `Фильтр` перед загрузкой расширенных данных: бумагу можно исключать навсегда или временно на 7 дней через `moex_bonds_config.py`;
- автоматические бизнес-правила фильтра: если до погашения меньше года — исключение навсегда; если амортизация уже началась (дата амортизации в прошлом) — исключение навсегда; если до ближайшей амортизации меньше года — исключение навсегда; если до оферты меньше года — исключение на 7 дней.
- временное исключение (7 дней) хранится в состоянии фильтра и не продлевается на каждом запуске автоматически; после истечения срока бумага снова попадает в опрос.
- бумаги с признаком `Тип облигации = Структурная облигация` автоматически исключаются навсегда и в следующих запусках больше не опрашиваются.
- структурные бумаги исключаются из итогового `bonds_traded` даже если выявлены уже на этапе расширенной загрузки карточки выпуска.
- в консоль и лог выводится статистика, сколько облигаций отфильтровано по каждому правилу.

### 2) `moex_bonds_config.py`
Простой файл настроек на русском языке.

Можно менять без аргументов запуска:
- количество потоков;
- таймауты и ретраи;
- срок жизни кэша;
- количество чанков;
- лимит облигаций для тестового запуска.

### 3) `generate_raw_links.py`
Служебный скрипт для генерации ссылок на raw-файлы GitHub.

## Какие файлы формируются

### Основной Excel
- `output/moex_bonds_full_export.xlsx`
  - `bonds_traded` (единый лист: торговые + описательные поля, без дублей и без лишних столбцов)
  - `data_quality` (контроль заполненности ключевых полей)
  - `column_dictionary` (словарь колонок простым языком прямо в Excel)

### Машиночитаемые справочники
- `output/moex_emitents.parquet`
- `output/moex_bond_coupons.parquet`
- `output/moex_bond_amortizations.parquet`
- `output/moex_bond_offers.parquet`
- `output/moex_emitents_history.csv` (история первого появления эмитентов в выгрузке)

### Технические данные
- Лог: `logs/moex_bonds_export.log` (перезаписывается на каждом запуске)
- Сырые JSON для отладки: `raw/` (очищается перед запуском)
- Кэш и checkpoint: `cache/moex/`
- Состояние фильтра исключений: `cache/moex/filter_state.json`

## Документация по столбцам
- Подробное описание колонок основного отчета: `docs/moex_bonds_full_export_columns.md`.
- В скрипте автоматически убираются дубли однотипных колонок, удаляются лишние поля из бизнес-списка и приводятся названия к русскому виду.

## Как запускать через VS Code
1. Откройте проект в VS Code.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. При необходимости отредактируйте `moex_bonds_config.py`.
4. Запустите `moex_bonds_export.py` кнопкой **Run Python File**.

---

## Что важно знать
- Если запуск оборвался из-за сети, скрипт продолжит с checkpoint.
- Лог не «пухнет»: каждый запуск полностью затирает старый файл.
- Сырые данные в `raw/` автоматически очищаются на старте.
- Если кэш стал слишком большим, скрипт автоматически удаляет самые старые кэш-файлы.
- В конце запуска скрипт выводит общее время выполнения.
