# Скрининг облигаций — каркас проекта

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Где менять настройки

Все параметры находятся в `config.py`:
- пути к папкам;
- лимиты и таймауты;
- параметры кэша;
- параметры логирования;
- базовые правила отбора.

## Что делает `main.py`

- создаёт рабочие папки;
- проверяет корректность настроек;
- запускает конвейер обработки;
- сохраняет данные в SQLite (`/db/bonds.sqlite3`) с WAL-режимом;
- создаёт итоговый Excel-файл в `/output`;
- выводит итоговую сводку в консоль.

## Что делает `erase_data.py`

```bash
python erase_data.py
```

Скрипт полностью очищает папки:
- `/db`
- `/cache`
- `/raw`
- `/logs`

Используйте, если нужно сбросить данные и начать заново.

## Где смотреть результаты

- Excel: `/output/bonds_screening_result.xlsx`
- Логи: `/logs/app.log` (+ ротационные архивы)

## Структура хранения

- `/db` — база SQLite с обработанными записями;
- `/raw` — сырые данные от источников (перезаписываемые);
- `/cache` — кэш и чекпоинт состояния (`state.json`), безопасно удаляется;
- `/output` — итоговые файлы для пользователя;
- `/logs` — журналы выполнения.

## Типовые проблемы

1. `Ошибка конфигурации` при старте.
   - Проверьте значения в `config.py` (таймауты, лимиты, пути).
2. Скрипт завершился с критической ошибкой.
   - Откройте `/logs/app.log` и посмотрите последние строки.
3. Нужен полный сброс после неудачного запуска.
   - Выполните `python erase_data.py`, затем `python main.py`.
4. Нет итогового Excel.
   - Проверьте ошибки в логах и наличие папки `/output`.
5. Похоже, что данные «зависли» на старом состоянии.
   - Удалите кэш через `erase_data.py` или вручную очистите `/cache`.
