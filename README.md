# Проект-скелет для скрининга облигаций

Этот проект подготовлен как базовый каркас под Windows 10/11 и Python 3.11+.
Все текстовые файлы проекта используют UTF-8.

## Структура проекта

- `main.py` — базовая точка входа, создаёт рабочие папки и подтверждает инициализацию.
- `erase_data.py` — утилита очистки содержимого папок `db`, `cache`, `raw`, `logs`.
- `config.py` — централизованные настройки путей и параметров.
- `docs/` — место для дополнительной документации.
- `cache/` — временные кэш-данные.
- `db/` — база данных проекта.
- `output/` — итоговые файлы (например, Excel-отчёты).
- `raw/` — сырые входные данные.
- `logs/` — логи работы скриптов.

## Установка зависимостей

```bash
pip install -r requirements.txt
```

На текущем этапе внешние зависимости не требуются, поэтому `requirements.txt` пустой.

## Запуск проекта

```bash
python main.py
```

После запуска будет создана базовая структура папок (если чего-то не хватает) и выведено сообщение:

`Проект инициализирован`

## Очистка служебных данных

```bash
python erase_data.py
```

Что делает скрипт:

- очищает только содержимое папок `db`, `cache`, `raw`, `logs`;
- сами папки сохраняет;
- если папки не существует, создаёт её;
- проверяет безопасность пути (чтобы не удалять файлы вне корня проекта);
- при ошибках удаления (файл занят, нет прав) пишет предупреждение и продолжает работу;
- в конце выводит статистику удалений и итоговый статус `SUCCESS` или `WARN`.

## Где смотреть результаты и логи

- Итоговые файлы размещайте в `output/`.
- Логи работы размещайте в `logs/`.

## Что хранится в папках и что можно удалять

- `db/` — база данных (обычно не удаляется без необходимости).
- `raw/` — сырые загруженные данные (можно очищать/пересоздавать).
- `cache/` — временные кэш-артефакты (безопасно очищать).
- `logs/` — файлы логов (безопасно очищать).

Для полного сброса служебных данных используйте `erase_data.py`.
