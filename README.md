# Сбор облигаций MOEX в Excel

## Что делает проект
Скрипт получает данные по облигациям с MOEX, фильтрует выпуски по сроку до погашения, обогащает данные информацией об эмитенте и формирует Excel-отчёт.

Сценарий запускается напрямую из VS Code: **без `argparse` и без параметров командной строки**.

---

## Основной файл
- `moex_bonds_to_excel.py` — главный скрипт выгрузки.

## Что происходит по шагам
1. Создаются папки `logs`, `raw`, `cache`, `output`.
2. Папка `raw` очищается перед запуском.
3. Лог `logs/moex_bonds.log` перезаписывается (всегда только последний запуск).
4. Читается основной кэш выгрузки.
5. Запрашиваются свежие бумаги с MOEX.
6. Дополнительно парсится блок рыночных данных: объём торгов (`VOLTODAY`), оборот (`VALTODAY`), число сделок (`NUMTRADES`) и биржевая доходность (`YIELD`).
7. Исключаются бумаги, у которых до погашения меньше 1 года.
8. Загружаются суточные метрики и отмечаются бумаги, где амортизация уже началась или начнётся менее чем через 1 год.
9. Для всех бумаг после фильтра по погашению выполняется проверка оферт через **ДОХОД** и **Corpbonds**.
10. Результаты этой проверки сохраняются в отдельный кэш на **7 дней** (`cache/offer_verification_cache.json`).
11. Для ускорения этапа проверки оферт используются отдельные быстрые таймауты/повторы и расширенная многопоточность.
12. Если карточка Corpbonds не найдена по ISIN (404), скрипт автоматически пробует адрес с `SECID`.
13. После проверки повторно применяется фильтр: до даты оферты должно быть больше 1 года.
14. Выполняется проверка качества данных.
12. Добавляются данные эмитента (`ISSUER_NAME`, `ISSUER_INN`, `ISSUER_BOND_CLASS`), признак `QUALIFIED_INVESTOR` и тип облигации `BOND_TYPE`.
13. Структурные облигации полностью исключаются из итогового отчёта и помечаются в пожизненном кэше, чтобы не определять этот тип заново на каждом запуске.
14. Если `COUPONPERIOD = 0`, период пытается восстановиться из `COUPONFREQUENCY` и запоминается в пожизненном кэше.
15. Если после восстановления `COUPONPERIOD` всё равно `0`, такая бумага исключается из итогового отчёта.
16. Оферты из MOEX используются как дополнительный источник (fallback), если внешние источники временно не дали ответ.
17. В отчёте учитываются только **актуальные** оферты (с датой сегодня или в будущем). Прошедшие оферты не считаются активными.
18. Если у бумаги несколько дат оферты из источников, берётся ближайшая актуальная дата.
19. Для данных эмитента используется **пожизненный кэш**.
20. Даты в итоге приводятся к формату `ДД.ММ.ГГГГ`.
21. Сохраняются `raw/moex_bonds_raw.json` и `output/moex_bonds.xlsx`.
22. В лог записывается время выполнения.
23. Перед сохранением автоматически удаляются устаревшие колонки из старого кэша (например, ранее удалённые поля классификации эмитента).

## Что в итоговом Excel
### Лист `MOEX_BONDS`
- `ISIN` — в первой колонке.
- Добавлены столбцы-разделители между блоками данных. Название каждой группы теперь пишется вертикально прямо в разделителе (объединение ячеек 1-2 строки), чтобы при сворачивании макет оставался читаемым.
- Теперь каждый блок колонок идёт подряд и имеет свою кнопку сворачивания/разворачивания. Разделители сделаны шире и окрашиваются разными оттенками, чтобы соседние блоки визуально не сливались:
  - `Эмитент`: `ISSUER_NAME`, `ISSUER_INN`, `ISSUER_BOND_CLASS`, `SHORTNAME`.
  - `Квалификация и тип`: `QUALIFIED_INVESTOR`, `BOND_TYPE`.
  - `Оферты`: `HAS_PUT_CALL_OFFER`, `PUT_CALL_OFFER_DATE`.
  - `Погашение и амортизация`: `HAS_AMORTIZATION`, `AMORTIZATION_START_DATE`, `MATDATE`.
  - `Купоны`: `COUPONVALUE`, `ACCRUEDINT`, `COUPONPERIOD`, `COUPONPERCENT`.
  - `Рынок`: `FACEVALUE`, `FACEUNIT`, `PRIMARYBOARDID`, `PREVLEGALCLOSEPRICE`, `PREVPRICE`, `VOLTODAY`, `VALTODAY`, `NUMTRADES`, `YIELD`.
- `QUALIFIED_INVESTOR` — ✔/✖.
- `ISSUER_BOND_CLASS` — тип бумаги на рынке облигаций (государственный, корпоративный, муниципальный/субфедеральный, иностранный).
- `BOND_TYPE` — тип облигации (например, фикс/флоатер). Структурные облигации в итоговый лист не попадают.
- `HAS_PUT_CALL_OFFER` — тип оферты: `PUT`, `Call` или `✖`.
- `PUT_CALL_OFFER_DATE` — ближайшая дата оферты (по ДОХОД + Corpbonds, при недоступности — fallback на MOEX).
- `HAS_AMORTIZATION` — ✔/✖ (финальное погашение не считается амортизацией).
- `AMORTIZATION_START_DATE` — дата старта амортизации (если есть).
- `ACCRUEDINT` — НКД.
- `MATDATE` — дата погашения (формат `ДД.ММ.ГГГГ`).
- `VOLTODAY` — объём торгов за день (количество бумаг).
- `VALTODAY` — оборот торгов за день (в деньгах).
- `NUMTRADES` — количество сделок за день.
- `YIELD` — доходность к погашению по данным MOEX (в % годовых).

Проверка для контроля: для `RU000A106RR9` поле `YIELD` должно быть `14.94` (по данным биржи на момент запроса).
- `SECID` скрыт (для технических связок).
- Колонки `SECNAME`, `LISTLEVEL`, `STATUS` удалены.

## Структура папок
- `logs/` — лог последнего запуска.
- `raw/` — сырые данные для отладки (очищается автоматически).
- `cache/` — кэши (основной, пожизненный по эмитентам и `COUPONPERIOD`, суточный по метрикам, а также отдельный 7-дневный кэш проверки оферт `offer_verification_cache.json`).
- `output/` — итоговый Excel.

---

## Как запустить
```bash
python moex_bonds_to_excel.py
```

После запуска проверьте:
- `output/moex_bonds.xlsx`
- `logs/moex_bonds.log`

## Новый расчёт плавающего купона (COUPONPERCENT)
- Скрипт находит выпуски, где `COUPONVALUE = 0`.
- Для таких бумаг сначала запрашивается страница `analytics.dohod.ru/bond/<ISIN>` и читается блок **«Привязка к индексу»**.
- Если формулу не удалось получить через ДОХОД, включается fallback на `corpbonds.ru/bond/<ISIN>`.
- Запросы к ДОХОД и fallback к CORPBONDS выполняются **многопоточно** (параллельно), чтобы ускорить обработку большого числа флоатеров.
- Для нестабильной сети добавлены повторы запросов к обоим источникам: это снижает риск пропуска формулы из-за временных ошибок.
- Поддержаны формулы вида:
  - `CBR_RATE + X`
  - `RUONIA + X`
  - `G_CURVE_RUS + X`
  - `∑KC + X%` (из CORPBONDS, `KC/КС` автоматически приводится к ключевой ставке ЦБ)
  - `∑RUONIA + X%` (из CORPBONDS)
- Если в описании формулы есть фраза про срок (например, «сроком погашения 7 лет»), для `G_CURVE_RUS` автоматически берётся нужный срок из текста.
- Актуальные базовые индексы собираются автоматически:
  - `CBR_RATE` — из таблицы ключевой ставки ЦБ РФ.
  - `RUONIA` — из таблицы RUONIA ЦБ РФ.
  - `G_CURVE_RUS` — из MOEX ZCYC (`yearyields`) по нужному сроку.
- Расчётные значения `COUPONPERCENT` подсвечиваются **жёлтым** в Excel как приблизительные (получены расчётом, а не прямым полем биржи).
- Кэш расчётов хранится 7 дней в `cache/coupon_formula_cache.json`.
- Если MOEX позже отдаст реальный `COUPONPERCENT` (> 0), расчётный кэш по этой бумаге удаляется и значение берётся из MOEX.
