# Скрипт выгрузки всех торгуемых облигаций MOEX

Скрипт собирает все облигации с Московской биржи (фильтр `is_traded=1`), сохраняет результат в Excel и считает динамику цены относительно предыдущего запуска.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Что делает скрипт

1. Загружает список всех торгуемых облигаций MOEX.
2. Догружает карточки бумаг (дефолт, техдефолт, квалифицированный инвестор, тип облигации, эмитент).
3. Загружает данные эмитентов (название и ИНН).
4. Загружает историю объёмов за последние 20 календарных дней и суммирует по каждой бумаге.
5. Загружает амортизационные графики и вычисляет дату начала амортизации.
6. Сравнивает текущую цену с предыдущей выгрузкой из SQLite.
7. Формирует Excel `output/MoexBonds.xlsx` (в режиме отладки файл всегда перезаписывается).
8. В консоли отображаются прогресс-бары по этапам, а при остановке Ctrl+C в лог пишется событие остановки.

## Где менять настройки

Все параметры находятся в `config.py`:

- пути к папкам;
- таймауты, ретраи, backoff;
- параллельность запросов;
- имя итогового Excel;
- режим `DEBUG_SINGLE_EXCEL_FILE`.

## Выходные папки

- `output/` — итоговый Excel-файл;
- `logs/` — лог-файлы с ротацией;
- `db/` — база SQLite с предыдущими ценами (для расчёта динамики);
- `cache/` — состояние прогресса `state.json` и кэш;
- `raw/` — резерв под сырые данные;
- `docs/` — документация.

## Сброс артефактов

Полная очистка папок `db`, `cache`, `raw`, `logs`:

```bash
python erase_data.py
```

## Частые проблемы

1. `ModuleNotFoundError` — не установлены зависимости (`pip install -r requirements.txt`).
2. Пустой/обрезанный Excel — временная ошибка сети или недоступность MOEX API, проверьте `logs/app.log`.
3. Долго работает — уменьшите `MAX_CONCURRENT_TASKS` или запустите позже (пиковая нагрузка API).
4. Нет динамики цены — это первый запуск или в БД ещё нет прошлых цен.
5. Пустые поля рейтинга у части выпусков — MOEX ISS API заполняет рейтинг и его описание не для всех бумаг; скрипт автоматически подставляет найденные поля `*RATING*`, `*RATING_DATE*`, `*RATING_DESCRIPTION*`, а при отсутствии оставляет пусто.
6. При ручной остановке Ctrl+C проверьте `logs/app.log` — там будет запись об остановке пользователем и этап, на котором был запуск.
7. `IndentationError` или другая синтаксическая ошибка — файл мог быть повреждён при копировании. Проверьте командой `python -m py_compile main.py app/*.py config.py erase_data.py` и перезапишите файлы проекта в UTF-8.
