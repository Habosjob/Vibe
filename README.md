# Vibe — утилиты для работы с данными облигаций MOEX

## Что делает проект

### 1) `moex_bonds_export.py`
Главный скрипт, который собирает данные по облигациям MOEX и формирует готовые файлы.

Сейчас логика такая:
- берутся только торгуемые облигации;
- листы `bonds_traded` и `bond_descriptions` объединяются в один итоговый лист `bonds_traded`;
- дубли полей при объединении автоматически удаляются;
- заголовки торговых колонок переименовываются на понятные русские названия;
- справочники сохраняются в формате **Parquet** (быстрее и удобнее для машинной обработки, чем Excel);
- справочник `moex_emitents` обновляется только новыми эмитентами (старые строки не перезаписываются);
- расширенные данные грузятся параллельно, чанками, с checkpoint и кэшем.

### 2) `moex_bonds_config.py`
Простой файл настроек на русском языке.

Можно менять без аргументов запуска:
- количество потоков;
- таймауты и ретраи;
- срок жизни кэша;
- количество чанков;
- лимит облигаций для тестового запуска.

### 3) `generate_raw_links.py`
Служебный скрипт для генерации ссылок на raw-файлы GitHub.

---

## Какие файлы формируются

### Основной Excel
- `output/moex_bonds_full_export.xlsx`
  - `bonds_traded` (единый лист: торговые + описательные поля, без дублей)

### Машиночитаемые справочники
- `output/moex_emitents.parquet`
- `output/moex_bond_coupons.parquet`
- `output/moex_bond_amortizations.parquet`
- `output/moex_bond_offers.parquet`

### Технические данные
- Лог: `logs/moex_bonds_export.log` (перезаписывается на каждом запуске)
- Сырые JSON для отладки: `raw/` (очищается перед запуском)
- Кэш и checkpoint: `cache/moex/`

---

## Как запускать через VS Code
1. Откройте проект в VS Code.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. При необходимости отредактируйте `moex_bonds_config.py`.
4. Запустите `moex_bonds_export.py` кнопкой **Run Python File**.

---

## Что важно знать
- Если запуск оборвался из-за сети, скрипт продолжит с checkpoint.
- Лог не «пухнет»: каждый запуск полностью затирает старый файл.
- Сырые данные в `raw/` автоматически очищаются на старте.
- В конце запуска скрипт выводит общее время выполнения.
