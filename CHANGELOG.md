# CHANGELOG

## 2026-02-25
- Исправлена заливка столбцов-разделителей: цвет теперь применяется ко всей высоте столбца, а не только к верхним строкам.
- Вернута корректная типизация данных для Excel: даты выгружаются как даты Excel с отображением `DD.MM.YYYY`, что включает группировку фильтров по годам (в т.ч. `NEXTCOUPON`).
- Обновлены форматы чисел: для целых используется `# ##0`, для дробных — `# ##0.00` (включая `ISSUESIZE` и `ISSUESIZEPLACED`).
- Актуализированы тесты экспорта Excel/CSV под новую логику форматирования.

## 2026-02-25
- По обратной связи возвращён полный парсинг полей MOEX: убрано ограничение `securities.columns`, теперь сохраняются все колонки из `securities`.
- Экспорт Excel/CSV переведён на динамические колонки: в файлы попадают все поля, которые реально пришли от MOEX.
- Обновлены тесты на параметры запроса и экспорт дополнительных колонок.

## 2026-02-25
- Оптимизирован запрос к MOEX ISS: добавлен `iss.only=securities` и фиксированный набор `securities.columns`, чтобы не скачивать лишние блоки и поля.
- Улучшен Excel-экспорт: автоширина колонок, фиксированная шапка, фильтры, контрастная шапка и зебра-раскраска строк для лучшей читаемости.
- Добавлены pytest-тесты на параметры запроса MOEX и форматирование итогового Excel.

## 2026-02-25
- Исправлен экспорт: по умолчанию результат сохраняется в `output/moex_bonds.xlsx`; для `.csv` добавлен UTF-8 BOM для корректной кириллицы в Excel.
- Обновлены тесты на запись `.xlsx`/`.csv` и валидацию неподдерживаемого формата.
- Добавлен локальный скрипт `run.py` для загрузки всех облигаций MOEX в CSV.
- Реализованы: ретраи, задержки между запросами, логирование, raw-дампы с автоочисткой.
- Добавлены pytest-тесты для пагинации/ретраев и очистки raw-хранилища.
- Добавлена документация в `docs/`.
- Исправлен бесконечный цикл пагинации в MOEX-клиенте: добавлена защита от повторяющихся страниц и от ответа, где MOEX отдает весь список за один запрос.
- Добавлены тесты, проверяющие остановку при повторе страниц и при непагинируемом ответе MOEX.
