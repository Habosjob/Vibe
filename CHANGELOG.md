# Changelog

## 2026-02-21
- Добавлен новый конфиг `moex_bonds_config.py` с русскими комментариями по каждому параметру запуска и отдельными сроками жизни кэша по типам данных.
- Реализовано деление загрузки расширенных данных на 10 чанков (`CHUNK_COUNT=10`).
- Добавлен механизм checkpoint: после каждого чанка сохраняется прогресс, что позволяет продолжать работу после сбоев сети.
- Основной Excel-файл разгружен: `bond_boards` исключён, а справочники `emitents`, `bond_coupons`, `bond_amortizations`, `bond_offers` вынесены в отдельные Excel-файлы.
- Исправлена читаемость `bond_descriptions`: добавен широкий формат (1 строка на 1 облигацию вместо длинного вертикального списка полей).
- Улучшено оформление итоговых Excel: цветной заголовок, жирный шрифт, автофильтр, фиксация первой строки, автоширина колонок.
- Обновлена русскоязычная документация в `README.md`.
- Обновлен `IMPROVEMENTS.md`: реализованные идеи перенесены в changelog, добавлены новые приоритетные предложения.

## 2026-02-20
- Добавлен скрипт `moex_bonds_export.py` для выгрузки торгуемых облигаций MOEX, расширенных данных по выпускам и данных по эмитентам в Excel.
- Реализованы: логирование в `logs/` с перезаписью, очистка `raw/` на старте, кэширование в `cache/`, параллельная загрузка данных, вывод этапов и времени выполнения.
- Добавлен `requirements.txt` с актуальными зависимостями для запуска скрипта.
- Добавлена русскоязычная документация в `README.md`.
- Добавлены проектные файлы `IMPROVEMENTS.md` и `CHANGELOG.md`.
- Исправлена проблема «бесконечной» загрузки справочника: удалён массовый проход по всему `/iss/securities` для рынка облигаций.
- Добавлена точечная загрузка справочных данных эмитентов только по найденным `SECID` через `q=<SECID>` с параллельной обработкой и кэшем в `cache/moex/reference_rows/`.
- Исправлена очистка `raw/`: служебный файл `.gitkeep` больше не удаляется.
- Исправлена агрегация эмитентов (устранён `KeyError` из-за регистра имён колонок).
