# CHANGELOG

## 2026-02-26
- Добавлены совместимые заглушки `_resolve_secondary_identifier` и `_fetch_with_fallback`, чтобы частично обновленные сборки не падали с `AttributeError` и продолжали ISIN-only запросы.
- Добавлен защитный helper `_is_payload_empty` для совместимости со старыми версиями fallback-логики ДОХОД; это устраняет `NameError` при частично обновленном окружении/файлах.
- Уточнен режим источника ДОХОД: этап обогащения снова работает строго по `ISIN` (бумаги без `ISIN` пропускаются на этом этапе), fallback на `SECID` убран как некорректный для данного источника.
- Улучшено сопоставление меток в карточке ДОХОД (нестрогое распознавание заголовков с вариациями пунктуации/пробелов), что возвращает извлечение цены/индекса/даты при измененной подписи строк.
- Усилен парсер карточек ДОХОД: извлечение меток/значений теперь работает по строкам таблицы (`tr/td/th`) и корректно обрабатывает вложенные теги (`span/div`), что возвращает заполнение купонов/оферт при изменившейся верстке страницы.
- Исправлено обогащение `COUPONPERCENT` через ДОХОД: теперь поле дозаполняется не только при пустой строке, но и при маркерах отсутствия значения (`—`, `N/A`, `None`) и при нулевом купоне, если доступна индексная формула.
- Исправлено заполнение оферты из ДОХОД: `OFFERDATE` теперь обновляется и при пустом описании ближайшего события (если дата YTM не совпадает с погашением), что возвращает пропущенные оферты.
- Добавлена статистика реальных изменений от ДОХОД (`добавлено/обновлено` для `RealPrice`, `COUPONPERCENT`, `OFFERDATE`) в консольную сводку, `SUMMARY` и run-metrics.
- Добавлены pytest-тесты на новые правила дозаполнения купона/оферты и на счётчики изменений ДОХОД.
- Исправлено повторное использование пустого кэша ДОХОД: если в checkpoint у бумаги нет ни цены, ни индекса, ни даты/события YTM, выполняется повторный запрос вместо cache-hit.
