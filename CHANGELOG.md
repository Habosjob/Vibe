# Changelog

## 2026-02-18 (обновление 13)

### Исправлено
- Исправлена проблема «призрачной» колонки `ISSUER_BOND_CLASS` в Excel после отката: теперь перед сохранением данные очищаются от устаревших полей из старого кэша.

### Изменено
- Добавлена служебная очистка удалённых колонок (`ISSUER_BOND_CLASS`, `ISSUER_SECTOR`, `SECTORID`, `INSTRID`) перед сохранением кэша, raw и Excel.
- Обновлён `README.md` с пояснением автоматической очистки старых колонок.

## 2026-02-18 (обновление 12)

### Изменено
- По запросу пользователя полностью удалены поля классификации эмитента: `ISSUER_BOND_CLASS` (и ранее удалённый `ISSUER_SECTOR`) больше не формируются и не попадают в итоговый Excel.
- В группе `Эмитент` оставлены только `ISSUER_NAME`, `ISSUER_INN`, `SHORTNAME`.
- Упрощён запрос к MOEX: убраны неиспользуемые поля `SECTORID` и `INSTRID`.

### Улучшено
- Отчёт возвращён к более стабильному и понятному составу колонок без спорной классификации.

## 2026-02-18 (обновление 11)

### Добавлено
- Добавлено поле `ISSUER_BOND_CLASS` для типа бумаги (государственный, корпоративный, муниципальный/субфедеральный, иностранный).

### Изменено
- Поле `ISSUER_SECTOR` удалено из итоговой выгрузки по запросу пользователя.
- В группе `Эмитент` и на листе `INFO` оставлено только поле `ISSUER_BOND_CLASS` как единый источник классификации типа бумаги.
- Обновлены `README.md` и описание колонок в Excel.

### Улучшено
- Упрощена структура отчёта: исключено дублирование и спорная эвристика отраслей по названию эмитента.

## 2026-02-17 (обновление 10)

### Добавлено
- Добавлен парсинг объёма торгов из блока `marketdata`: теперь в итоговую таблицу попадают поля `VOLTODAY` (объём), `VALTODAY` (оборот) и `NUMTRADES` (количество сделок).
- Лист `INFO` дополнен понятными описаниями новых рыночных полей.

### Изменено
- Группа `Рынок` на листе `MOEX_BONDS` расширена новыми колонками ликвидности.
- Обновлён `README.md`: добавлено описание этапа парсинга объёма торгов и новых колонок в отчёте.

### Улучшено
- Реализована часть идеи по анализу ликвидности: собраны базовые метрики торгов, на основе которых можно добавить фильтр минимального объёма в следующих обновлениях.

## 2026-02-17 (обновление 9)

### Изменено
- Подписи групп перенесены в столбцы-разделители с вертикальным текстом и объединением ячеек 1-2 строки: это улучшает читаемость в сложенном виде.
- Столбцы-разделители расширены, чтобы названия групп не выглядели сжатыми.
- Для разделителей добавлена палитра из разных оттенков, чтобы соседние блоки визуально не сливались.

## 2026-02-17 (обновление 8)

### Добавлено
- Добавлены отдельные столбцы-разделители между группами колонок на листе `MOEX_BONDS`.
- Названия групп перенесены в столбцы-разделители (по макету), чтобы блоки легче читались визуально.

### Изменено
- Настройка outline для Excel пересобрана: каждая группа получает собственную кнопку складывания/раскладывания.

## 2026-02-17 (обновление 7)

- Убрано объединение ячеек в строке подписей групп (именно оно приводило к конфликтам структуры листа в части случаев).

### Изменено

## 2026-02-17 (обновление 6)

### Добавлено
- Добавлены новые группы колонок для сворачивания/разворачивания на листе `MOEX_BONDS`: `Квалификация и тип`, `Оферты`, `Погашение и амортизация`, `Купоны`, `Рынок` (к ранее существующей группе `Эмитент`).

- Обновлён `README.md` с понятным описанием всех групп колонок и их назначения.

## 2026-02-17 (обновление 5)

### Добавлено
- Добавлен отдельный фильтр по дате оферты: бумаги исключаются, если `PUT_CALL_OFFER_DATE` наступает раньше, чем через 1 год от текущей даты.
- В служебные поля (и суточный кэш) добавлены признаки `EXCLUDE_BY_OFFER_DATE` и `OFFER_DATE_EXCLUDE_REASON`.

### Изменено
- Логика этапов обновлена: фильтр по офертам выполняется сразу после фильтра амортизации и до обогащения эмитентами.
- Обновлён `README.md` с описанием нового правила исключения облигаций по дате оферты.

## 2026-02-17 (обновление 4)

### Исправлено
- Структурные облигации теперь действительно исключаются из итоговой выгрузки (добавлен отдельный фильтр после обогащения эмитентами).
- Исправлен расчёт `HAS_PUT_CALL_OFFER`: учитываются только актуальные оферты (с датой сегодня/в будущем). Прошедшие оферты больше не считаются активными.
- Исправлен выбор `PUT_CALL_OFFER_DATE` при нескольких офертах: используется ближайшая актуальная дата.
- Для пожизненного кэша эмитентов добавлен флаг структурного типа `secid_to_is_structural`, чтобы не переопределять его на каждом запуске.

### Изменено
- Обновлена версия схемы суточного кэша до `v6` для пересчёта оферт по новой логике.
- Обновлён `README.md` (актуальные правила по структурным бумагам и офертам).

## 2026-02-17 (обновление 3)

### Исправлено
- Усилена нормализация `BOND_TYPE`: теперь любые варианты, содержащие «структур... облигац...», приводятся к `Не указан`.
- Добавлен отдельный фильтр `COUPONPERIOD`: если после восстановления период остался `0`, бумага исключается из выгрузки.
- Исправлен парсинг оферт: теперь `HAS_PUT_CALL_OFFER` определяется по фактическому наличию записей в `offers`, а не только по английским значениям типа оферты.
- Исправлен выбор даты оферты: берётся ближайшая будущая дата, а если таких нет — последняя доступная историческая.

### Изменено
- Обновлена версия схемы суточного кэша до `v5` для автоматического пересчёта полей оферт по новым правилам.
- Обновлён `README.md` с пояснениями новой логики по `COUPONPERIOD` и офертам.

## 2026-02-17 (обновление 2)

### Добавлено
- Добавлены поля `HAS_PUT_CALL_OFFER` и `PUT_CALL_OFFER_DATE` с парсингом из блока `offers` MOEX bondization.
- В суточный кэш добавлено хранение признака и даты Put/Call оферт.
- В пожизненный кэш эмитентов добавлено хранение исправленного `COUPONPERIOD` для выпусков, где API возвращает `0`.

### Изменено
- Для `BOND_TYPE` добавлена нормализация: значения типа `Структурная облигация` теперь заменяются на `Не указан` и запоминаются, чтобы повторно не опрашивать API.
- Для выпусков с `COUPONPERIOD = 0` теперь период вычисляется по `COUPONFREQUENCY`, сохраняется в пожизненном кэше и повторно не пересчитывается.
- Обновлён лист `INFO`, README и порядок колонок в Excel с учётом полей Put/Call оферт.

### Улучшено
- Реализована идея из improvement по расширению анализа оферт: в текущем релизе добавлен автоматический сбор признака/даты оферты и подготовлена база для календаря оферт.

## 2026-02-17

### Добавлено
- Добавлено поле `BOND_TYPE` (тип облигации: фикс, флоатер и т.д.) в итоговую таблицу и лист `INFO`.
- Добавлены служебные поля маркировки для фильтра амортизации: `EXCLUDE_BY_AMORTIZATION` и `AMORTIZATION_EXCLUDE_REASON` (для внутренней логики и кэша).

### Изменено

- Форматы дат `MATDATE` и `AMORTIZATION_START_DATE` в Excel переведены в вид `ДД.ММ.ГГГГ`.
- Реализован фильтр: бумаги, где амортизация уже началась или начнётся менее чем через год, исключаются из выгрузки.
- Суточный кэш метрик теперь хранит флаг исключения по амортизации, чтобы не перерасходовать запросы на такие бумаги.
- Улучшен сбор MOEX: добавлена безопасная постраничная логика с остановкой при отсутствии новых ключей.

### Улучшено
- Реализована идея из `IMPROVEMENTS.md` №3: загрузка всех страниц MOEX ISS с защитой от зацикливания API.

### Добавлено

- Добавлен фильтр: бумаги с датой погашения менее 1 года исключаются до этапов обогащения.
- Добавлены поля `HAS_AMORTIZATION` (✔/✖) и `AMORTIZATION_START_DATE`.
- Добавлено поле `ACCRUEDINT` (НКД) в итоговую таблицу.
- Добавлен суточный кэш `cache/daily_security_metrics_cache.json` для полей амортизации (обновление раз в 24 часа).

- Обновлено описание листа `INFO` и порядок колонок в Excel с учётом новых полей.

### Улучшено
- Добавлено версионирование суточного кэша амортизации (`schema_version`) для автоматического пересчёта после изменения правил парсинга.
- Обновлены этапы логирования (7 шагов) для более понятного отображения прогресса.
