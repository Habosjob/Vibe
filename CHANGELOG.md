# Changelog

## 2026-02-21
- В листе `bonds_traded` добавлена группировка колонок с поддержкой сворачивания/разворачивания в Excel (outline).
- Добавлены цветные столбцы-разделители с названиями групп, чтобы блоки полей визуально не сливались.
- Обновлены `README.md` и `docs/moex_bonds_full_export_columns.md` с понятным описанием новой структуры отчета.

## 2026-02-21
- Добавлена валидация обязательных колонок из API MOEX: при изменении структуры теперь выводится понятная ошибка вместо «тихой» поломки отчета.
- Добавлен контроль размера кэша (`CACHE_SOFT_LIMIT_MB`): при превышении лимита удаляются самые старые кэш-файлы.
- Добавлен файл `output/moex_emitents_history.csv` с историей первого появления эмитентов.
- В основной Excel добавлен новый лист `column_dictionary` с простыми русскими пояснениями по колонкам.
- Усилено объединение таблиц: дубли колонок теперь склеиваются стабильнее (с учетом регистра и лишних пробелов).
- Из итогового листа `bonds_traded` удалены лишние столбцы по обновленному бизнес-списку.
- Реализованные пункты удалены из `IMPROVEMENTS.md`; добавлены новые идеи следующего этапа.

## 2026-02-21
- Добавлена отдельная русскоязычная документация `docs/moex_bonds_full_export_columns.md` с пояснением столбцов листов `bonds_traded` и `data_quality`.
- Расширен перевод технических англоязычных колонок в `moex_bonds_export.py`, чтобы в итоговом Excel оставалось меньше необработанных названий.
- Усилена дедупликация колонок: добавлены алиасы полей из карточки бумаги и объединение дублей по названиям с сохранением непустых значений.
- README дополнен ссылкой на новую документацию по столбцам и описанием логики удаления дублей.

## 2026-02-21
- Выполнено объединение листов `bonds_traded` и `bond_descriptions` в единый лист `bonds_traded`.
- Реализовано удаление дублей полей при слиянии торговых данных и карточки выпуска.
- Добавлено переименование технических торговых заголовков в понятные русские названия в итоговом листе `bonds_traded`.
- Обновление справочника `moex_emitents` переведено в инкрементальный режим: существующие эмитенты не перезаписываются, добавляются только новые.
- Справочники `moex_emitents`, `moex_bond_amortizations`, `moex_bond_coupons`, `moex_bond_offers` переведены из Excel в формат Parquet для более быстрого машинного чтения.
- Добавлена повторная загрузка только проблемных `SECID` после основного прохода для повышения полноты выгрузки.
- Добавлена автоматическая архивация `raw/*.json` после успешного запуска с хранением ограниченного числа архивов.
- В основной Excel добавлен лист `data_quality` с контролем заполненности ключевых полей.
- Обновлены `README.md`, `requirements.txt`, `IMPROVEMENTS.md` и комментарии в конфигурации.

## 2026-02-20
- Добавлен скрипт `moex_bonds_export.py` для выгрузки торгуемых облигаций MOEX, расширенных данных по выпускам и данных по эмитентам в Excel.
- Реализованы: логирование в `logs/` с перезаписью, очистка `raw/` на старте, кэширование в `cache/`, параллельная загрузка данных, вывод этапов и времени выполнения.
- Добавлен `requirements.txt` с актуальными зависимостями для запуска скрипта.
- Добавлена русскоязычная документация в `README.md`.
- Добавлены проектные файлы `IMPROVEMENTS.md` и `CHANGELOG.md`.
- Исправлена проблема «бесконечной» загрузки справочника: удалён массовый проход по всему `/iss/securities` для рынка облигаций.
- Добавлена точечная загрузка справочных данных эмитентов только по найденным `SECID` через `q=<SECID>` с параллельной обработкой и кэшем в `cache/moex/reference_rows/`.
- Исправлена очистка `raw/`: служебный файл `.gitkeep` больше не удаляется.
- Исправлена агрегация эмитентов (устранён `KeyError` из-за регистра имён колонок).
