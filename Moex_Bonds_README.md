# Moex_Bonds — быстрая выгрузка облигаций MOEX в Excel

## Что делает скрипт
`Moex_Bonds.py` автоматически:
1. Скачивает CSV с витрины MOEX (ISS) по URL.
2. Пропускает служебные строки и берёт таблицу, начиная с заголовка `SECID`.
3. Приводит типы данных (даты и числовые колонки).
4. Сохраняет результат в `xlsx` через **ускоренный движок `xlsxwriter`**.
5. Применяет оформление для читабельности:
   - цветной заголовок,
   - фильтры,
   - закрепление первой строки,
   - автоширина колонок,
   - форматы чисел/дат,
   - чередование цвета строк.
6. Пишет подробный **перезаписываемый** лог (достаточный для дебага).
7. Показывает интерактивный прогресс в консоли (прогресс-бар + «живой» статус на длинном шаге экспорта).

---

## Ключевые улучшения производительности
В новой версии ускорение достигается в основном за счёт:

- перехода с `openpyxl` на `xlsxwriter` для записи Excel (без режима `constant_memory`, чтобы гарантировать корректное заполнение всех колонок);
- отказа от медленного поячейчного форматирования (циклы по всем строкам/ячейкам);
- применения форматирования «пакетно» на уровне колонок и условного форматирования листа;
- облегчённой автоконвертации числовых столбцов.

На больших таблицах это обычно даёт ускорение в разы (цель: минимум x3).

## Запуск
```bash
python3 Moex_Bonds.py
```

Пример с настройками:
```bash
python3 Moex_Bonds.py \
  --output bonds_today.xlsx \
  --sheet BONDS \
  --log Moex_Bonds.log \
  --timeout 90
```

---

## Что и где настраивается
Все основные параметры — через аргументы командной строки:

- `--url` — источник CSV (по умолчанию уже задан URL MOEX для облигаций).
- `--output` — имя/путь итогового Excel-файла.
- `--sheet` — имя листа Excel.
- `--timeout` — таймаут HTTP-запроса.
- `--log` — путь к лог-файлу.
  - По умолчанию имя формируется от имени скрипта (`Moex_Bonds.log`).
  - Для других скриптов используйте отдельные имена логов, например `--log Other_Script.log`.

Пример:
```bash
python3 Moex_Bonds.py --url "https://..." --output result.xlsx --log my_script.log
```

## Интерактивный вывод в консоли
Скрипт показывает прогресс-бар по этапам:
1. Загрузка CSV
2. Очистка данных
3. Конвертация типов
4. Экспорт в Excel
5. Завершение

Внутри шага 4 дополнительно показывается «живой» статус (спиннер) на длительных операциях, чтобы было видно, что процесс идёт.

---

## Логи и дебаг
- Лог-файл ведётся через `logging.FileHandler(..., mode="w")`.
- Это значит, что при каждом новом запуске лог **перезаписывается**.
- Формат лога: время, уровень, имя логгера, сообщение.
- При ошибке пишется полный stack trace.
- Логируются ключевые этапы и длительности (включая время сохранения Excel и общее время выполнения).

Рекомендация: для каждого отдельного скрипта задавайте свой `--log`.
