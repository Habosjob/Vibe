# Moex_Bonds — выгрузка облигаций MOEX в Excel

## Что делает скрипт
`Moex_Bonds.py` автоматически:
1. Скачивает CSV с витрины MOEX (ISS) по URL.
2. Пропускает служебные строки и берёт таблицу, начиная с заголовка `SECID`.
3. Приводит типы данных (даты и числовые колонки).
4. Сохраняет результат в `xlsx`.
5. Применяет оформление для читабельности:
   - цветной заголовок,
   - фильтры,
   - закрепление первой строки,
   - автоширина колонок,
   - форматы чисел/дат,
   - чередование цвета строк.
6. Пишет подробный **перезаписываемый** лог и показывает прогресс-бар в консоли.
7. Внутри долгого шага форматирования Excel показывает «живой» статус, чтобы было видно, что скрипт не завис.

---

## Запуск
```bash
python3 Moex_Bonds.py
```

Пример с настройками:
```bash
python3 Moex_Bonds.py \
  --output bonds_today.xlsx \
  --sheet BONDS \
  --log Moex_Bonds.log \
  --timeout 90
```

---

## Что и где настраивается
Все основные параметры — через аргументы командной строки:

- `--url` — источник CSV (по умолчанию уже задан URL MOEX для облигаций).
- `--output` — имя/путь итогового Excel-файла.
- `--sheet` — имя листа Excel.
- `--timeout` — таймаут HTTP-запроса.
- `--log` — путь к лог-файлу.
  - По умолчанию имя формируется от имени скрипта (`Moex_Bonds.log`).
  - Для других скриптов используйте отдельные имена логов, например `--log Other_Script.log`.

Пример:
```bash
python3 Moex_Bonds.py --url "https://..." --output result.xlsx --log my_script.log
```

---

## Логи и дебаг
- Лог-файл ведётся через `logging.FileHandler(..., mode="w")`.
- Это значит, что при каждом новом запуске лог **перезаписывается**.
- Формат лога содержит время, уровень, имя логгера и сообщение.
- При ошибке в лог пишется полный stack trace.
- Дополнительно логируются длительности ключевых этапов (стилизация и сохранение Excel) — это помогает в диагностике «подвисаний».

Рекомендация: для каждого отдельного скрипта задавайте свой `--log` (например, `Moex_Bonds.log`).

## Интерактивность в консоли
Скрипт выводит прогресс-бар по этапам:
1. Загрузка CSV
2. Очистка данных
3. Конвертация типов
4. Сохранение Excel
5. Завершение

Во время долгой стилизации внутри шага 4 дополнительно печатается промежуточный статус вида:

```text
⏳ Шаг 4/5: Стилизация строк 1200/8000 (15.0%)
```

Это позволяет видеть, что выполнение идёт и скрипт не завис даже на больших таблицах.
