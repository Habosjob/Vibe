# Moex_Bonds — выгрузка облигаций MOEX в Excel

## Что делает скрипт
`Moex_Bonds.py` автоматически:
1. Загружает CSV с витрины MOEX ISS.
2. Использует кеш CSV при актуальном TTL.
3. Удаляет служебные/лишние колонки по списку из YAML.
4. Удаляет полностью пустые колонки.
5. Переносит указанные колонки в конец таблицы.
6. Автоконвертирует даты и числовые поля, но сохраняет выбранные поля как текст.
7. Экспортирует данные в Excel с таблицей, полосатой заливкой и цветовой шкалой для выбранных колонок.
8. Показывает интерактивный прогресс в консоли (цветной прогресс-бар + спиннер).
9. Пишет перезаписываемый лог для дебага.

---

## Структура
- `config/moex_bonds.yaml` — все настройки запуска и обработки.
- `docs/Moex_Bonds.md` — документация по функционалу.
- `logs/Moex_Bonds.log` — лог запуска (перезаписывается каждый запуск).
- `logs/cache/moex_rates.csv` — кеш CSV.
- `logs/cache/moex_rates_state.json` — состояние последней выгрузки (hash данных).

## Настройки YAML

### `source`
- `url` — URL CSV MOEX.

### `network`
- `timeout_sec` — таймаут HTTP.

### `output`
- `excel_path` — путь итогового Excel.
- `sheet_name` — имя листа.
- `drop_columns` — список колонок, которые удаляются перед обработкой.
  - В текущей конфигурации дополнительно удаляются: `DURATION`, `NAME`, `OFFERDATE`, `YIELDATWAP`.
- `heatmap_columns` — точные имена колонок для цветовой шкалы (без масок).
  - Важно: теперь сравнение **точное**, поэтому `COUPONDATE` не окрашивается, если указана только `COUPON`.
- `text_columns` — колонки, которые принудительно сохраняются как текст.
  - В текущей конфигурации: `INN`.
- `move_to_end_columns` — колонки, которые переносятся в конец таблицы.
  - В текущей конфигурации: `WAPRICE`.

### `logging`
- `path` — путь к логу. Лог открывается в режиме `mode="w"` (перезапись).

### `cache`
- `csv_path` — файл кеша CSV.
- `state_path` — файл состояния выгрузки (hash/метаданные).
- `ttl_sec` — время жизни кеша CSV.

### `performance`
- `width_sample_rows` — сколько строк брать для расчёта ширины колонок (меньше = быстрее).
- `skip_rebuild_if_unchanged` — если `true`, пересборка Excel пропускается при неизменных данных.

## Логирование
Лог содержит:
- источник данных (кеш/сеть),
- список удалённых колонок,
- факт переноса колонок в конец,
- длительность экспорта,
- полный stack trace при ошибке.

Формат строки лога:
`YYYY-MM-DD HH:MM:SS | LEVEL | Moex_Bonds | message`
