# Moex_Bonds — выгрузка облигаций MOEX в Excel

## Что делает скрипт
`Moex_Bonds.py` автоматически:
1. Загружает CSV с витрины MOEX ISS.
2. Использует кеш CSV при актуальном TTL.
3. Удаляет полностью пустые колонки.
4. Автоконвертирует даты и числовые поля.
5. Экспортирует данные в Excel.
6. Применяет «цветастое» оформление: таблица со стилем, полосы, цветовые шкалы по доходностям.
7. Показывает интерактивный прогресс в консоли (цветной прогресс-бар + спиннер).
8. Пишет перезаписываемый лог для дебага.

---

## Структура
- `config/moex_bonds.yaml` — все настройки.
- `docs/Moex_Bonds.md` — документация.
- `logs/Moex_Bonds.log` — лог запуска (перезаписывается).
- `logs/cache/moex_rates.csv` — кеш CSV.
- `logs/cache/moex_rates_state.json` — состояние последней выгрузки (hash данных).

---

## Ключевое ускорение повторного запуска
Добавлен режим **пропуска пересборки Excel**, если данные не изменились:
- считается hash табличной части CSV;
- hash сохраняется в `logs/cache/moex_rates_state.json`;
- если на следующем запуске hash тот же и файл Excel уже есть, пересборка листа пропускается.

Именно этот шаг убирает самые дорогие операции (`xlsxwriter`) и резко ускоряет повторный прогон.

---

## Запуск
```bash
python3 Moex_Bonds.py --config config/moex_bonds.yaml
```

## Настройки YAML

### `source`
- `url` — URL CSV MOEX.

### `network`
- `timeout_sec` — таймаут HTTP.

### `output`
- `excel_path` — путь итогового Excel.
- `sheet_name` — имя листа.
- `heatmap_columns` — список масок колонок (без учета регистра), для которых добавляется цветовая шкала (красный → жёлтый → зелёный).

### `logging`
- `path` — путь к логу. Лог всегда открывается в режиме `mode="w"`.

### `cache`
- `csv_path` — файл кеша CSV.
- `state_path` — файл состояния выгрузки (hash/метаданные).
- `ttl_sec` — время жизни кеша CSV.

### `performance`
- `width_sample_rows` — сколько строк брать для расчёта ширины колонок (меньше = быстрее).
- `skip_rebuild_if_unchanged` — если `true`, пропускает пересборку Excel при неизменных данных.

---

## Логирование
Лог содержит:
- источник данных (кеш/сеть),
- размер таблицы,
- длительность экспорта,
- полный stack trace при ошибке.

Формат строки лога:
`YYYY-MM-DD HH:MM:SS | LEVEL | Moex_Bonds | message`
