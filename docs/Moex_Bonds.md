# Moex_Bonds — выгрузка облигаций MOEX в Excel

## Что делает скрипт
`Moex_Bonds.py` автоматически:
1. Загружает CSV с витрины MOEX ISS.
2. Может использовать локальный кеш CSV (ускоряет повторный запуск).
3. Очищает данные и удаляет полностью пустые колонки.
4. Автоматически определяет даты и числовые поля.
5. Экспортирует результат в Excel (`xlsxwriter`).
6. Применяет визуальное оформление (цветной заголовок, фильтр, freeze panes, полосы строк, форматы чисел/дат).
7. Показывает интерактивный прогресс в консоли (цветной прогресс-бар + спиннер).
8. Пишет перезаписываемый лог в `logs/`.

---

## Структура папок
- `docs/` — документация по скриптам.
- `logs/` — логи и кеши выполнения.
- `config/moex_bonds.yaml` — все настройки запуска.

---

## Запуск
```bash
python3 Moex_Bonds.py
```

Явно указать конфиг:
```bash
python3 Moex_Bonds.py --config config/moex_bonds.yaml
```

---

## Настройки (YAML)
Все рабочие параметры вынесены в `config/moex_bonds.yaml`:

- `source.url` — URL источника CSV MOEX.
- `network.timeout_sec` — HTTP-таймаут.
- `output.excel_path` — путь к итоговому Excel.
- `output.sheet_name` — имя листа.
- `logging.path` — путь к логу (перезаписывается на каждом запуске).
- `cache.csv_path` — путь кеш-файла CSV.
- `cache.ttl_sec` — сколько секунд кеш считать актуальным.
- `performance.width_sample_rows` — сколько строк анализировать для автоширины колонок (меньше = быстрее).

---

## Как ускорено
1. Повторные запуски читают CSV из кеша (`logs/cache/moex_rates.csv`) при актуальном TTL.
2. Форматирование Excel выполняется пакетно (без поячейчной обработки).
3. Автоширина колонок считается по ограниченной выборке строк.

---

## Логирование и дебаг
- Формат: `дата | уровень | логгер | сообщение`.
- Режим перезаписи: каждый запуск даёт свежий лог.
- В лог пишутся ключевые этапы, длительности и исключения со стеком.
