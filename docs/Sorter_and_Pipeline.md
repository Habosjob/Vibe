# Python_Sorter + main.py — фильтрация и общий запуск пайплайна

## Обновленная логика (без `Moex_Bonds_Filtered.xlsx`)

### `Python_Sorter.py`
Скрипт читает `Moex_Bonds.xlsx`, применяет фильтры и теперь работает так:
1. Загружает текущий реестр исключений `DropedBonds.csv` (если файл есть).
2. **До фильтрации** исключает бумаги из `Moex_Bonds`, которые уже есть в `DropedBonds` и относятся к **активным** фильтрам.
3. Применяет активные фильтры к оставшимся бумагам.
4. Перезаписывает `Moex_Bonds.xlsx` (новый файл `Moex_Bonds_Filtered.xlsx` больше не используется).
   - Сохранение идет через тот же форматтер, что и в `Moex_Bonds.py` (таблица, стили, ширины, цветовые шкалы), поэтому оформление не теряется.
5. Обновляет `DropedBonds.csv`, добавляя только новые исключенные бумаги и не создавая дублей.

Итог:
- уже отсортированные бумаги не гоняются повторно через фильтрацию и запись в основной Excel;
- пока `DropedBonds.csv` существует и фильтр включен — эти бумаги автоматически пропускаются;
- если удалить `DropedBonds.csv`, реестр исключений начнется заново.

> Важно: если фильтр отключить (`enabled: false`), бумаги, исключенные этим фильтром ранее, перестают считаться «активно исключенными» и снова могут попасть в итоговый `Moex_Bonds.xlsx`.

### `main.py`
Оркестратор пайплайна без изменений по шагам:
1. Запускает `Moex_Bonds.py`.
2. Затем запускает `Python_Sorter.py`.

Запуск:
```bash
python3 main.py --config config/moex_bonds.yaml
```

## Логи (перезаписываемые)
Каждый скрипт пишет отдельный лог в режиме перезаписи (`mode="w"`):
- `logs/Moex_Bonds.log`
- `logs/Python_Sorter.log`
- `logs/main.log`

Логов достаточно для дебага полного сценария по шагам.

## Настройка через YAML
Все настройки вынесены в `config/moex_bonds.yaml`.
Для форматирования итогового `Moex_Bonds.xlsx` сортировщик использует общие секции `output.*` и `performance.width_sample_rows` из этого же конфига.

### `pipeline`
```yaml
pipeline:
  logging:
    path: "logs/main.log"
```

### `sorter`
```yaml
sorter:
  input:
    excel_path: "Moex_Bonds.xlsx"
    sheet_name: "MOEX_BONDS"
  output:
    excel_path: "Moex_Bonds.xlsx"
    sheet_name: "MOEX_BONDS"
    dropped_path: "DropedBonds.csv"
    dropped_encoding: "utf-8-sig"
  logging:
    path: "logs/Python_Sorter.log"
  filters:
    - name: "exclude_structured_bonds"
      enabled: true
      column: "BOND_TYPE"
      equals: "Структурная облигация"
      reason: "BOND_TYPE = Структурная облигация"
```

### Как включать/выключать фильтры
У каждого фильтра есть флаг `enabled`:
- `true` — фильтр активен;
- `false` — фильтр отключен (и записи этого фильтра из `DropedBonds` не используются для пред-исключения).

---

## Отладка
Если фильтрация отрабатывает не так, как ожидается:
1. Проверьте точное имя колонки в `column`.
2. Проверьте значение `equals`.
3. Проверьте `enabled`.
4. Откройте `logs/Python_Sorter.log`:
   - видно, сколько записей загружено из `DropedBonds`;
   - сколько бумаг исключено как «ранее отброшенные»;
   - сколько новых бумаг исключено фильтрами;
   - сколько строк записано в итоговый `Moex_Bonds.xlsx`.
