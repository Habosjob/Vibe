# Документация по столбцам `output/moex_bonds_full_export.xlsx`

## Для чего нужен этот файл
`moex_bonds_full_export.xlsx` — основной Excel-отчет по торгуемым облигациям MOEX в «читаемом» формате для пользователя.

В файле 3 листа:
- `bonds_traded` — ключевые данные по облигациям (без технического «шума»);
- `data_quality` — проверка заполненности важных полей;
- `column_dictionary` — словарь колонок простыми словами.

---

## Лист `bonds_traded`: группы и разделители

Чтобы отчетом было удобно пользоваться даже при большом числе колонок:
- столбцы разбиты на логические группы по жизненному циклу облигации;
- перед каждой группой добавлен отдельный цветной столбец-разделитель с названием группы;
- в Excel включено сворачивание/разворачивание групп по кнопкам структуры (outline).

Текущие подписи разделителей в `bonds_traded`:
- `1) Идентификация выпуска`;
- `2) Эмитент`;
- `3) Параметры выпуска и номинала`;
- `4) Купонный доход`;
- `5) Погашение и оферты`;
- `6) Рыночные данные и ликвидность`.

Цвета у разделителей чередуются, чтобы при сворачивании группы не сливались визуально.

## Лист `bonds_traded`: основные столбцы

> Важно: часть колонок специально удаляется по бизнес-правилам, чтобы отчет был компактнее и проще.

| Столбец | Что означает простыми словами |
|---|---|
| Код бумаги | Уникальный код облигации на бирже (SECID). |
| Краткое наименование | Короткое имя бумаги для удобного просмотра. |
| ISIN | Международный код бумаги. |
| Эмитент | Название компании, выпустившей облигацию. |
| ИНН эмитента | Налоговый номер эмитента. |
| Валюта номинала | Валюта, в которой указан номинал. |
| Номинал | Номинальная стоимость одной облигации. |
| Дата погашения | Когда эмитент должен вернуть номинал. |
| Дата следующего купона | Ближайшая дата купонной выплаты. |
| Ставка купона, % | Процентная ставка купонного дохода. |
| Размер купона | Сумма одного купонного платежа. |
| НКД | Накопленный купонный доход на текущую дату. |
| Доходность по средневзвешенной цене | Расчетная доходность исходя из средневзвешенной цены. |
| Цена предыдущей сделки | Цена последней сделки за предыдущий торговый день. |
| Средневзвешенная цена предыдущего дня | Средняя цена за прошлый день с учетом объема. |
| Количество сделок | Сколько сделок по бумаге было за день. |
| Объем за день | Количество бумаг в сделках за день. |
| Оборот за день | Денежный оборот по бумаге за день. |
| Дата оферты | Дата ближайшей оферты (если есть). |
| Цена оферты | Цена, по которой может быть исполнена оферта. |

## Лист `data_quality`

| Столбец | Что означает |
|---|---|
| Поле | Проверяемый столбец из `bonds_traded`. |
| Всего строк | Сколько всего облигаций в выгрузке. |
| Пустых значений | Сколько записей не заполнено по этому полю. |
| Заполнено, % | Процент заполненности поля. |

## Лист `column_dictionary`
На этом листе дублируется список колонок `bonds_traded` и дается короткое пояснение по каждой колонке. Это нужно, чтобы файл можно было передать человеку без технической подготовки.

---

## Как скрипт борется с дублями колонок
Скрипт объединяет дубли колонок не только по точному названию, но и с учетом:
- лишних пробелов;
- различий в регистре;
- синонимов из карточки выпуска (например, `ISIN` и `ISIN код`).

При объединении сохраняется первое непустое значение, чтобы не терять полезные данные.
