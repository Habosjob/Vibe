# Документация по столбцам `output/moex_bonds_full_export.xlsx`

## Для чего нужен этот файл
`moex_bonds_full_export.xlsx` — это основной Excel-отчет по торгуемым облигациям MOEX.

В файле 2 листа:
- `bonds_traded` — все основные данные по облигациям;
- `data_quality` — контроль заполненности ключевых полей.

---

## Лист `bonds_traded`: описание основных столбцов

| Столбец | Что означает простыми словами |
|---|---|
| Код бумаги | Уникальный код облигации на бирже (SECID). |
| Краткое наименование | Короткое имя бумаги для удобного просмотра. |
| Полное наименование | Полное официальное название выпуска. |
| Латинское наименование | Название бумаги на латинице. |
| ISIN | Международный код бумаги. |
| Регистрационный номер | Номер регистрации выпуска. |
| Уровень листинга | Уровень допуска бумаги к торгам. |
| Валюта номинала | Валюта, в которой указан номинал облигации. |
| Номинал | Номинальная стоимость одной облигации. |
| Дата погашения | Когда эмитент должен вернуть номинал. |
| Ставка купона, % | Процентная ставка купонного дохода. |
| Размер купона | Сумма одного купонного платежа. |
| Частота купона | Сколько раз в год платится купон. |
| Дата следующего купона | Ближайшая дата купонной выплаты. |
| Цена предыдущей сделки | Цена последней сделки за предыдущий торговый день. |
| Средневзвешенная цена предыдущего дня | Средняя цена за прошлый день с учетом объема. |
| Доходность по средневзвешенной цене | Расчетная доходность исходя из средневзвешенной цены. |
| НКД | Накопленный купонный доход на текущую дату. |
| Лот | Сколько бумаг в одном лоте. |
| Стоимость лота | Ориентировочная стоимость одного лота. |
| Цена оферты | Цена, по которой может быть исполнена оферта. |
| Дата оферты | Дата ближайшей оферты (если есть). |
| Торгуется | Признак, что бумага реально торгуется. |
| Статус | Биржевой статус бумаги (например, активна). |
| Код эмитента | Внутренний идентификатор компании-эмитента. |
| Эмитент | Название компании, выпустившей облигацию. |
| ИНН эмитента | Налоговый номер эмитента. |
| ОКПО эмитента | Код ОКПО эмитента. |
| Тип инструмента | Тип торгового инструмента. |
| Группа инструментов | Группа, к которой относится инструмент. |
| Основной режим торгов | Основной режим, где торгуется бумага. |
| Режим цены рынка | Режим, из которого берется рыночная цена. |

## Лист `data_quality`

| Столбец | Что означает |
|---|---|
| Поле | Проверяемый столбец из `bonds_traded`. |
| Всего строк | Сколько всего облигаций в выгрузке. |
| Пустых значений | Сколько записей не заполнено по этому полю. |
| Заполнено, % | Процент заполненности поля. |

---

## Что сделано для борьбы с дублями столбцов

В выгрузке автоматически убираются дубли, которые часто приходят из разных источников MOEX (торговые блоки + карточка выпуска), например:
- `ISIN` и `ISIN код`;
- `Код бумаги` и `Код ценной бумаги`;
- `Краткое наименование` и `Наименование ценной бумаги`;
- `Латинское наименование` и `Английское наименование`;
- `Номинал` и `Номинальная стоимость`.

Если у дублей есть разные заполненные значения, скрипт аккуратно объединяет их: в итоговом столбце сохраняется первое непустое значение.
