# Скрипт `main.py` — загрузка MOEX rates в SQLite

## Что делает скрипт
1. Загружает CSV с MOEX по ссылке из `config.py`.
2. Сохраняет файл в папки:
   - `raw/moex_rates.csv` (основной «сырой» файл),
   - `cache/moex_rates_cache.csv` (кэш-копия).
3. Обновляет SQLite-базу `DB/moex_rates.sqlite3` (таблица `rates`) **полной перезаливкой**.
4. Использует TTL кэша `12 часов`:
   - если данные свежие, в сеть не ходит;
   - если TTL истек, удаляет старые CSV и загружает новые.
5. Создает Excel-срез на 5 случайных строк с уникальными `SECID` в `BaseSnapshots/rates_snapshot.xlsx`.
6. Пишет технический лог в `logs/main.log` (файл перезаписывается каждый запуск).

## Запуск
```bash
python main.py
```

> Без CLI/argparse: просто запуск `main.py`.

## Что выводится в консоль
В консоли только этапы + прогресс-бары + финальный summary по времени.

## Настройка
Все настройки находятся в `config.py`.
Каждый параметр в файле снабжен комментарием: что делает, какие значения допустимы, значение по умолчанию.

Ключевые параметры:
- `CACHE_TTL_HOURS` — срок жизни кэша в часах.
- `REQUEST_TIMEOUT_SECONDS` — таймаут HTTP.
- `SOURCE_CSV_URL` — источник данных.
- Имена/пути папок и файлов (`RAW_DIR`, `CACHE_DIR`, `DB_DIR`, `LOGS_DIR` и т.д.).

## Оптимизация SQLite
При подключении применяются PRAGMA:
- `journal_mode=WAL`
- `synchronous=NORMAL`
- `temp_store=MEMORY`
- `cache_size=-200000`
- `mmap_size=1073741824`

Это ускоряет массовую перезаливку таблицы.

## Примечания по формату CSV MOEX
В исходном файле есть служебные/пустые строки перед заголовком.
Скрипт ищет строку заголовков, начинающуюся с `SECID`, и корректно пропускает «мусорные» строки.
