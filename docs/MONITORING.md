# Мониторинг и логи

## Логи
- Файл: `logs/latest.log`
- Перезаписывается при каждом запуске.
- Уровни:
  - `INFO` — этапы работы.
  - `WARNING/ERROR` — проблемы API и обработки.

## Прогресс в консоли
- Во время запуска выводятся этапы вида `[Этап X/9] ...`.
- Для длинных операций печатается текущий прогресс:
  - загрузка списка бумаг (счетчик загруженных бумаг);
  - обогащение амортизаций (`обработано N/M`, процент и примерная оценка оставшегося времени).
- Если найден чекпоинт, явно показывается сообщение о возобновлении работы.

## Чекпоинты на случай падения сети
- Файлы чекпоинтов: `state/checkpoints/bonds_fetch.json` и `state/checkpoints/amortization.json`.
- При сбое сети следующая попытка запуска продолжит с последнего успешного шага.
- После успешного завершения этапов чекпоинты автоматически удаляются.

## Что смотреть при проблемах
1. Наличие записей `Ошибка запроса`.
2. Количество ошибок в финальной сводке.
3. Проверить интернет-соединение и доступ к `iss.moex.com`.
4. Проверить наличие `state/checkpoints/*.json`: если файлы есть, это нормальный признак незавершенного предыдущего запуска.


## История запусков
- Для SQLite-режима используйте `python scripts/show_runs.py --limit 10` (таблица) или `--format json`.
- Скрипт читает таблицу `runs` и показывает длительность, количество обработанных/отфильтрованных бумаг и ошибки.
